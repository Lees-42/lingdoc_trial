# 灵档（LingDoc）快速运行教程

> 本教程面向首次运行灵档项目的开发者，手把手完成从环境准备到前后端联调的全过程。

---

## 1. 环境准备

在启动项目前，请确保您的电脑已安装以下软件：

| 软件 | 最低版本 | 验证命令 | 说明 |
|------|---------|---------|------|
| JDK | 17+ | `java -version` | 后端运行环境 |
| Maven | 3.8+ | `mvn -v` | 后端构建工具 |
| Node.js | 18+ | `node -v` | 前端运行环境 |
| MySQL | 8.0+ | — | 业务数据库 |
| Redis | 5.0+ | — | 缓存与会话 |

### 1.1 快速检查

打开终端（Windows 建议使用 PowerShell），依次执行：

```powershell
java -version
mvn -v
node -v
npm -v
```

如果以上命令都能正常输出版本号，说明环境基本就绪。

### 1.2 启动 MySQL 和 Redis

确保 MySQL 服务和 Redis 服务已在本地正常运行：

- **MySQL**：通常通过系统服务或 Docker 启动
- **Redis**：进入 `redis-temp/` 目录（项目自带 Windows 版 Redis），执行：

```powershell
cd redis-temp
.\redis-server.exe redis.windows.conf
```

> Redis 窗口保持运行，不要关闭。

## 2. 初始化数据库

### 2.1 创建数据库

使用任意 MySQL 客户端（如 Navicat、DataGrip、MySQL Workbench 或命令行）连接本地 MySQL，执行：

```sql
CREATE DATABASE `ruoyi`
  DEFAULT CHARACTER SET utf8mb4
  COLLATE utf8mb4_general_ci;
```

### 2.2 导入 SQL 脚本

在项目根目录下，执行以下命令导入系统表、菜单、字典等初始数据：

```powershell
cd ruoyi-server

# Windows 命令行示例（请替换 root 和 your_password）
mysql -u root -p ruoyi < sql\ry_20260321.sql
```

> 如果不需要定时任务功能，可以跳过 `quartz.sql`。

---

## 3. 配置后端

### 3.1 修改数据库连接

打开文件：

```
ruoyi-server/ruoyi-admin/src/main/resources/application-druid.yml
```

修改 `master` 数据源配置为您本地的 MySQL 账号密码：

```yaml
spring:
  datasource:
    type: com.alibaba.druid.pool.DruidDataSource
    druid:
      master:
        url: jdbc:mysql://localhost:3306/ruoyi?useUnicode=true&characterEncoding=utf8&zeroDateTimeBehavior=convertToNull&useSSL=true&serverTimezone=GMT%2B8
        username: root
        password: your_password
```

### 3.2 修改 Redis 配置（如有必要）

打开文件：

```
ruoyi-server/ruoyi-admin/src/main/resources/application.yml
```

确认 Redis 配置与您的本地环境一致：

```yaml
spring:
  data:
    redis:
      host: localhost
      port: 6379
      password:   # 如果 Redis 设置了密码，请填写
```

## 4. 启动后端

### 方式一：IDEA 直接运行（推荐开发调试）

1. 使用 IntelliJ IDEA 打开 `ruoyi-server/` 目录
2. 找到 `ruoyi-admin/src/main/java/com/ruoyi/RuoYiApplication.java`
3. 右键 `RuoYiApplication`，选择 **Run 'RuoYiApplication'**
4. 当看到控制台输出 **"若依启动成功"** 时，后端启动完成

### 方式二：Maven 命令行打包运行

```powershell
cd ruoyi-server

# 打包（跳过测试以加快速度）
mvn clean package -DskipTests

# 运行生成的 jar 包
java -jar ruoyi-admin/target/ruoyi-admin.jar
```

### 验证后端是否启动成功

打开浏览器访问：

```
http://localhost:8080/captchaImage
```

如果能看到验证码图片，说明后端服务已正常运行。

---

## 5. 启动前端

### 5.1 安装依赖

在项目根目录（与 `package.json` 同级）打开终端，执行：

```powershell
npm install
```

> 如果网络较慢，可以使用 `npm install --registry=https://registry.npmmirror.com` 切换国内镜像。

### 5.2 启动开发服务器

```powershell
npm run dev
```

当看到类似以下输出时，前端启动成功：

```
VITE v6.x  ready in xxx ms

➜  Local:   http://localhost/
➜  Network: http://192.168.x.x:80/
```

## 7. 常见问题

### Q1：后端启动报错 "Communications link failure"
**原因**：MySQL 未启动，或连接配置错误。  
**解决**：检查 MySQL 服务状态，并确认 `application-druid.yml` 中的 `url`、`username`、`password` 正确无误。

### Q2：后端启动报错 "Connection refused" 与 Redis 相关
**原因**：Redis 未启动，或密码错误。  
**解决**：确保 Redis 服务已启动；如设置了密码，请在 `application.yml` 中填写。

### Q3：登录时提示 "请求访问：/login，认证失败，无法访问系统资源"
**原因**：后端安全配置可能阻止了跨域请求。  
**解决**：前端通过 Vite 代理访问后端，请确保浏览器地址栏是 `http://localhost`（而非 `127.0.0.1` 或 IP 地址）。

### Q4：侧边栏出现空白菜单项
**原因**：后端 `sys_menu` 中保留了前端已删除模块的菜单记录（如角色管理、菜单管理等）。  
**解决**：暂时忽略，或在 MySQL 中清理 `sys_menu` 表里对应的不需要的前端页面记录。

### Q5：8080 端口被占用
**原因**：其他程序占用了 8080 端口。  
**解决**：
1. 修改后端端口：编辑 `ruoyi-server/ruoyi-admin/src/main/resources/application.yml`
   ```yaml
   server:
     port: 8081
   ```
2. 同步修改前端代理：编辑根目录 `vite.config.js`，将 `baseUrl` 改为 `http://localhost:8081`
