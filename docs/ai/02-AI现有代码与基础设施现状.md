# AI 现有代码与基础设施现状

> 本文档对当前仓库中 AI 相关代码进行「存量盘点」，明确「已有什么」和「还缺什么」，为后续开发提供基准线。

---

## 1. 已存在的基础设施

### 1.1 数据库层

文件位置：`ruoyi-server/sql/07-ai-module-mysql.sql`

该脚本定义了完整的 RAG（检索增强生成）数据库 Schema，**尚未并入主库脚本 `ry_20260321.sql`**。

| 表名 | 用途 | 关键字段 |
|------|------|----------|
| `kb_knowledge_base` | 知识库配置 | `chunk_size`, `chunk_overlap`, `embedding_model`, `total_files`, `total_chunks` |
| `kb_document` | 上传文档 | `content_text`, `file_type`, `chunk_count`, `status`, `error_msg` |
| `kb_chunk` | 文本分块 | `chunk_text`（FULLTEXT 索引）, `token_count`, `char_count` |
| `kb_embedding` | 向量存储 | `vector_data`（BLOB）, `vector_dimension`（默认 1536）, `normalized` |
| `ai_chat_session` | 对话会话 | `model_name`, `kb_id`, `total_tokens_in/out`, `last_message_at` |
| `ai_chat_message` | 对话消息 | `role`, `content`, `retrieval_chunks`（JSON）, `latency_ms`, `status` |

**状态**：Schema 设计完整，具备 RAG 全链路表结构，但数据库中尚未创建。

| 缺口 | 影响 | 优先级 |
|------|------|--------|
| LLM 客户端接入层 | 无法调用任何大模型 API | P0 |

## 5. 依赖与构建现状

### 5.1 Maven 依赖

当前 `ruoyi-server/pom.xml` 及各模块 POM 中**无任何 AI 相关依赖**：

| 依赖类型 | 缺失示例 |
|----------|----------|
| LLM SDK | `openai-java`, `deeplearning4j`, `langchain4j` |
| Embedding | `langchain4j-embeddings`, 本地模型推理库 |
| 文档解析 | `Apache Tika`, `pdfbox`, `poi`（已有但不用于 AI） |
| 向量库 | `milvus-java-sdk`, `pgvector` JDBC 扩展 |

### 5.2 后端编译状态

当前后端代码使用了 **MyBatis-Plus** 注解（`@TableName`, `BaseMapper`, `ServiceImpl` 等），但 **POM 中未显式声明 MyBatis-Plus 依赖**。该依赖可能通过传递依赖引入，但存在编译风险，需关注。
