## 4. 共性问题

### 4.1 底层 AI 能力完全缺失

已在 `docs/ai/02-AI现有代码与基础设施现状.md` 中详细记录，此处汇总与两个功能直接相关的缺口：

| 底层能力 | 状态 | 影响 |
|---------|------|------|
| LLM 客户端接入层 | 🔴 不存在 | 无法调用任何大模型 API |
| 文档解析（PDF/Word → 文本） | 🔴 不存在 | 自动规整无法读取文件内容做分类判断 |
| Prompt 模板管理 | 🔴 不存在 | 两个功能都需要结构化 Prompt |
| 异步任务执行器 | 🔴 不存在 | `extractFields` 可能需要异步执行 |
| 错误重试机制 | 🔴 未定义 | AI 调用超时/失败后的处理策略缺失 |

### 4.2 两个功能依赖同一套 Prompt 工程

虽然业务场景不同，但自动规整和表格填写助手的 AI 调用都依赖以下共性能力：

1. **文件内容提取**：将 PDF/Word/Excel 转为纯文本供 AI 分析
2. **结构化输出**：要求 AI 返回 JSON 格式（而非自由文本）
3. **中文 Prompt 工程**：两个功能都需要中文场景下的精确 Prompt 设计

**建议**：AI 开发同学可先实现一个通用的 `AiClientService` + `DocumentParserService`，再分别对接两个业务功能。

### 4.3 错误处理策略未定义

FAST-004 §3.1 定义了错误码（5001~5004），但代码中：
- 没有任何地方捕获和处理 AI 调用异常
- 任务状态 `"4"`（失败）虽然定义了，但没有代码会将其置为失败
- 超时重试逻辑完全缺失
