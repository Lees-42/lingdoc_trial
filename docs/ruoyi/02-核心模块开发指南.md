# 核心模块开发指南

> 本文档针对当前项目中**实际保留**的模块（参数配置、字典管理、操作日志、个人中心），说明其代码结构、扩展方式及编码规范。

---

## 1. 编码规范（基线要求）

所有新增或修改的代码须遵循以下规范：

### 1.1 命名规范
- **组件文件**：`PascalCase`，如 `UserProfile.vue`
- **API 函数**：`verb-noun` 动词-名词模式，如 `listConfig`、`updateUserProfile`
- **变量**：语义化，避免 `a`、`b`、`tmp` 等无意义命名

```javascript
// PASS
const searchQuery = reactive({ pageNum: 1, pageSize: 10 })

// FAIL
const q = reactive({ p: 1, s: 10 })
```

### 1.2 不可变性原则
- 状态更新优先使用展开运算符或 `Object.assign`，避免直接修改原对象
- 数组操作使用 `slice`、`filter`、`map`，避免 `push`、`splice` 直接修改（性能场景除外且需注释说明）

```javascript
// PASS
const updatedList = [...userList, newUser]

// FAIL
userList.push(newUser)
```

### 1.3 错误处理
- API 调用需有 `try-catch` 或在 `.catch()` 中处理错误
- 用户操作失败须给出明确的 `ElMessage` / `ElNotification` 提示

## 2. 参数配置（`system/config`）

### 2.1 接口层
文件：`src/api/system/config.js`

```javascript
import request from '@/utils/request'

// 查询参数列表
export function listConfig(query) {
  return request({
    url: '/system/config/list',
    method: 'get',
    params: query
  })
}

// 新增参数配置
export function addConfig(data) {
  return request({
    url: '/system/config',
    method: 'post',
    data: data
  })
}

// 修改参数配置
export function updateConfig(data) {
  return request({
    url: '/system/config',
    method: 'put',
    data: data
  })
}

// 删除参数配置
export function delConfig(configId) {
  return request({
    url: '/system/config/' + configId,
    method: 'delete'
  })
}

// 刷新参数缓存
export function refreshCache() {
  return request({
    url: '/system/config/refreshCache',
    method: 'delete'
  })
}
```

### 2.2 开发模式
- 页面：`views/system/config/index.vue`
- 采用标准的 **查询表格 + 表单弹窗** 模式
- 扩展建议：若需增加新的参数类型，优先在后端 `sys_config` 表中扩展，前端表格增加列即可

## 3. 字典管理（`system/dict`）

### 3.1 模块结构

```
src/views/system/dict/
├── index.vue     # 字典类型列表（主列表）
├── detail.vue    # 字典类型详情/编辑弹窗
└── data.vue      # 字典数据列表（子页面）
```

### 3.2 联动路由设计
字典数据页通过 **动态路由** 实现跳转：

文件：`src/router/index.js`

```javascript
export const dynamicRoutes = [
  {
    path: '/system/dict-data',
    component: Layout,
    hidden: true,
    permissions: ['system:dict:list'],
    children: [
      {
        path: 'index/:dictId(\\d+)',
        component: () => import('@/views/system/dict/data'),
        name: 'Data',
        meta: { title: '字典数据', activeMenu: '/system/dict' }
      }
    ]
  }
]
```

跳转示例：

```javascript
import { useRouter } from 'vue-router'

const router = useRouter()

function handleToData(row) {
  router.push('/system/dict-data/index/' + row.dictId)
}
```

### 3.3 扩展建议
- 新增字典类型：在 `dict/index.vue` 中维护弹窗表单
- 新增字典数据：在 `dict/data.vue` 中维护，注意 `dictType` 参数从路由或父级传入
- 字典回显：使用 `selectDictLabel()` 或 `<dict-tag>` 组件

## 4. 操作日志（`monitor/operlog`）

### 4.1 接口层
文件：`src/api/monitor/operlog.js`

```javascript
import request from '@/utils/request'

// 查询操作日志列表
// 注：此处为项目遗留代码，按 verb-noun 规范应命名为 listOperlog(query)
export function list(query) {
  return request({
    url: '/monitor/operlog/list',
    method: 'get',
    params: query
  })
}

// 删除操作日志
export function delOperlog(operId) {
  return request({
    url: '/monitor/operlog/' + operId,
    method: 'delete'
  })
}

// 清空操作日志
export function cleanOperlog() {
  return request({
    url: '/monitor/operlog/clean',

```

### 4.2 开发模式
- 页面：`views/monitor/operlog/index.vue`（列表页）、`views/monitor/operlog/detail.vue`（详情页）
- 通常为 **只读列表 + 筛选条件**，支持导出 Excel
- 扩展建议：若需增加新的操作类型筛选，联系后端在 `sys_oper_log` 中扩展字段

## 5. 个人中心（`system/user/profile`）

### 5.1 模块结构

```
src/views/system/user/profile/
├── index.vue        # 个人中心容器（Tab 切换）
├── userInfo.vue     # 基本资料编辑
├── resetPwd.vue     # 修改密码
└── userAvatar.vue   # 头像上传/裁剪
```

### 5.2 路由配置
文件：`src/router/index.js`

```javascript
{
  path: '/user',
  component: Layout,
  hidden: true,
  redirect: 'noredirect',
  children: [
    {
      path: 'profile/:activeTab?',
      component: () => import('@/views/system/user/profile/index'),
      name: 'Profile',
      meta: { title: '个人中心', icon: 'user' }
    }
  ]
}
```

### 5.3 接口层
文件：`src/api/system/user.js`

```javascript
import request from '@/utils/request'

// 查询用户个人信息
export function getUserProfile() {
  return request({
    url: '/system/user/profile',
    method: 'get'
  })
}

// 修改用户个人信息
export function updateUserProfile(data) {
  return request({
    url: '/system/user/profile',
    method: 'put',
    data: data
  })
}

// 用户密码重置
export function updateUserPwd(oldPassword, newPassword) {
  const data = { oldPassword, newPassword }
  return request({
    url: '/system/user/profile/updatePwd',
    method: 'put',
    params: data
  })
}

// 用户头像上传
export function uploadAvatar(data) {
  return request({
    url: '/system/user/profile/avatar',
    method: 'post',
    data: data
  })
}
```

### 5.4 扩展建议
- 新增 Tab 页：在 `profile/index.vue` 中添加 `<el-tab-pane>`，新建对应子组件
- 新增用户信息字段：同步修改 `userInfo.vue` 表单和 `updateUserProfile` 的数据结构

## 6. 新增同类型模块的标准步骤

以新增 **系统公告** 为例（假设后端已提供 `/system/notice` 接口）：

1. **创建 API 文件**：`src/api/system/notice.js`，函数命名遵循 `listNotice` / `getNotice` / `addNotice` / `updateNotice` / `delNotice`
2. **创建页面目录**：`src/views/system/notice/index.vue`
3. **注册路由**：若后端菜单已配置，无需手动注册；若为隐藏路由（如详情页），在 `src/router/index.js` 的 `dynamicRoutes` 中添加
4. **添加权限标识**：与后端 `sys_menu` 中的 `perms` 字段保持一致，如 `system:notice:list`
5. **字典回显**：若涉及状态/类型，使用 `getDicts('sys_notice_status')` 加载字典
