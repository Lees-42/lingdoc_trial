# 后端迁移与启动指南

> 本文档说明如何将 `ruoyi-server/` 后端项目启动并与当前前端进行联调。

---

## 1. 后端目录结构

```
ruoyi-server/
├── ruoyi-admin/          # Web 服务入口（Spring Boot 启动模块）
├── ruoyi-common/         # 公共工具类、常量、异常定义
├── ruoyi-framework/      # 框架核心（AOP、拦截器、安全配置、数据源等）
├── ruoyi-generator/      # 代码生成器模块
├── ruoyi-quartz/         # 定时任务模块
├── ruoyi-system/         # 系统管理业务模块（用户、菜单、角色、字典等）
├── sql/                  # 数据库初始化脚本
│   ├── ry_20260321.sql   # 主数据库脚本
│   └── quartz.sql        # 定时任务表脚本
├── bin/                  # 打包/清理脚本（注意与项目根目录 `bin/` 区分）
├── doc/                  # 后端相关文档
├── pom.xml               # Maven 父 POM
├── ry.bat                # Windows 后端启动/停止脚本
└── ry.sh                 # Linux 后端启动/停止脚本
```

---

## 2. 环境要求

| 环境 | 版本 | 说明 |
|------|------|------|
| JDK | 17+ | `ruoyi-server/pom.xml` 中指定 `java.version=17` |
| MySQL | 8.0+ | 用于存储业务数据 |
| Redis | 5.0+ | 用于缓存、会话、Token |
| Maven | 3.8+ | 用于后端项目构建 |

## 3. 数据库初始化

### 3.1 创建数据库
登录 MySQL，执行以下命令创建数据库：

```sql
CREATE DATABASE `ruoyi` DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
```

### 3.2 导入 SQL 脚本

```bash
# 导入主数据库（含系统表、菜单、字典、参数等）
mysql -u root -p ruoyi < ruoyi-server/sql/ry_20260321.sql

# 如需使用定时任务，额外导入 quartz 表
mysql -u root -p ruoyi < ruoyi-server/sql/quartz.sql
```

### 3.3 修改数据库连接配置
打开 `ruoyi-server/ruoyi-admin/src/main/resources/application-druid.yml`，修改以下配置：

```yaml
spring:
  datasource:
    type: com.alibaba.druid.pool.DruidDataSource
    druid:
      master:
        url: jdbc:mysql://localhost:3306/ruoyi?useUnicode=true&characterEncoding=UTF-8&connectionCollation=utf8mb4_unicode_ci&zeroDateTimeBehavior=convertToNull&useSSL=true&serverTimezone=GMT%2B8
        username: root
        password: your_password
```

### 3.4 修改 Redis 配置（如需要）
打开 `ruoyi-server/ruoyi-admin/src/main/resources/application.yml`，确认 Redis 地址：

```yaml
spring:
  data:
    redis:
      host: localhost
      port: 6379
      password:        # 若 Redis 有密码，请填写
```

## 4. 后端启动方式

### 方式一：IDEA 直接运行（推荐开发调试）
1. 使用 IDEA 打开 `ruoyi-server/` 目录（或仅导入该 Maven 项目）
2. 找到 `ruoyi-admin/src/main/java/com/ruoyi/RuoYiApplication.java`
3. 右键执行 `main` 方法
4. 控制台输出 `若依启动成功` 即表示启动完成

### 方式二：Maven 命令行打包运行
```bash
cd ruoyi-server

# 打包（跳过测试以加快速度）
mvn clean package -DskipTests

# 运行生成的 jar 包
java -jar ruoyi-admin/target/ruoyi-admin.jar
```

### 方式三：使用启动脚本（Windows）
```bash
cd ruoyi-server

# 打包
mvn clean package -DskipTests

# 直接运行 jar（推荐开发调试）
java -jar ruoyi-admin/target/ruoyi-admin.jar

# 如需使用 ry.bat，需先将 jar 复制到根目录
# copy ruoyi-admin\target\ruoyi-admin.jar .
# ry.bat
# 按提示输入 1 启动，2 关闭，3 重启
```

### 方式四：使用启动脚本（Linux / macOS）
```bash
cd ruoyi-server

# 打包
mvn clean package -DskipTests

# 直接运行 jar（推荐开发调试）
java -jar ruoyi-admin/target/ruoyi-admin.jar

# 或使用 ry.sh（需确保 jar 与脚本在同一目录）
# cp ruoyi-admin/target/ruoyi-admin.jar .
# sh ry.sh start
# sh ry.sh status
# sh ry.sh stop
# sh ry.sh restart
```

## 5. 前后端联调

### 5.1 默认端口配置

| 服务 | 地址 | 说明 |
|------|------|------|
| 前端开发服务器 | `http://localhost:80` | `vite.config.js` 中配置 `server.port: 80` |
| 后端接口服务 | `http://localhost:8080` | `application.yml` 中配置 `server.port: 8080` |

### 5.2 前端代理配置
当前前端已通过 Vite 代理将 `/dev-api` 请求转发到后端：

文件：`vite.config.js`

```javascript
const baseUrl = 'http://localhost:8080' // 后端接口

server: {
  proxy: {
    '/dev-api': {
      target: baseUrl,
      changeOrigin: true,
      rewrite: (p) => p.replace(/^\/dev-api/, '')
    }
  }
}
```

前端环境变量 `.env.development`：
```
VITE_APP_BASE_API = '/dev-api'
```

### 5.3 联调步骤
1. 确保 MySQL 和 Redis 已启动
2. 启动后端：`RuoYiApplication` 或 `java -jar ruoyi-admin.jar`
3. 启动前端：`npm run dev`
4. 浏览器访问 `http://localhost`
5. 使用默认账号登录：`admin` / `admin123`

## 6. 后端端口修改（可选）

若 `8080` 端口被占用，需要**同时修改前后端配置**：

### 6.1 修改后端端口
编辑 `ruoyi-server/ruoyi-admin/src/main/resources/application.yml`：

```yaml
server:
  port: 8081    # 修改为未被占用的端口
```

### 6.2 修改前端代理
编辑根目录 `vite.config.js`：

```javascript
const baseUrl = 'http://localhost:8081' // 同步修改为后端新端口
```

## 7. 常见问题

### Q1：后端启动报错 "Communications link failure"
**原因**：MySQL 未启动或连接配置错误。  
**解决**：检查 MySQL 服务状态，并确认 `application-druid.yml` 中的 `url`、`username`、`password` 正确。

### Q2：后端启动报错 "Connection refused" 与 Redis 相关
**原因**：Redis 未启动或密码错误。  
**解决**：启动 Redis 服务，或检查 `application.yml` 中 Redis 配置。

### Q3：登录时提示 "请求访问：/login，认证失败，无法访问系统资源"
**原因**：后端 `ruoyi-framework` 中的跨域或安全配置阻止了前端请求。  
**解决**：前端通过 Vite 代理访问，确保浏览器地址栏是 `http://localhost`（非 `127.0.0.1` 或 IP 地址），因为 Vite 代理仅在开发服务器上生效。

### Q4：如何确认后端接口正常？
打开浏览器访问：
```
http://localhost:8080/captchaImage
```
若能返回验证码图片，说明后端服务已正常启动。
