# 定制变更与兼容记录

> 本文档记录当前项目与标准 RuoYi-Vue 3.9.2 分离版前端之间的差异，帮助开发者理解精简范围，并安全地进行二次开发。

---

## 1. 变更总览

当前项目是在 **RuoYi-Vue 3.9.2 分离版前端** 的基础上，删除了不常用模块后的产物。核心基础设施（路由、权限、请求、状态管理、布局）**完整保留**，但大量业务页面和对应 API 已被移除。

## 4. 二次开发注意事项

### 4.1 后端菜单与前端路由的兼容
若后端 `sys_menu` 表中仍保留了已删除模块的菜单数据（如角色管理、菜单管理等），前端 `permission.js` 会在 `loadView()` 中尝试加载对应的 `.vue` 文件。

**当前行为**：
- 若组件文件不存在，`loadView()` 返回 `undefined`
- Vue Router 在渲染时内容区将显示为空白页，并在浏览器控制台输出组件未找到的警告

**建议**：
1. 同步清理后端 `sys_menu` 中已删除模块的菜单记录
2. 或在前端 `filterAsyncRouter()` 中增加组件存在性校验（需自定义修改 `permission.js`）
3. 最稳妥的方式是保持前后端菜单数据一致

### 4.2 新增系统模块的标准位置
若需要新增一个系统管理模块（如"示例管理"），请按以下路径组织：

```
src/
├── api/
│   └── system/
│       └── example.js       # API 封装
├── views/
│   └── system/
│       └── example/
│           └── index.vue    # 列表页面
└── router/
    └── index.js             # 若为隐藏路由，在 dynamicRoutes 中注册
```

后端菜单的 `component` 字段应填写：`system/example/index`

### 4.3 恢复已删除模块的方法
若业务需要恢复某个已删除的标准模块（如角色管理），建议从官方仓库完整复制以下文件：

1. **页面文件**：`views/模块路径/` 下的所有 `.vue` 文件
2. **API 文件**：`src/api/模块路径.js`
3. **字典配置**：若模块使用了特定字典，需在 `sys_dict_type` / `sys_dict_data` 中补充
4. **路由/权限**：若后端菜单已存在，通常无需额外配置；若为纯前端隐藏路由，复制 `router/index.js` 中的对应 `dynamicRoutes`

> 官方仓库地址：`https://gitee.com/y_project/RuoYi-Vue`

### 4.4 依赖包说明
当前 `package.json` 仍保留了原 RuoYi 项目的全部依赖（如 `echarts`、`vue-cropper` 等）。即使某些页面已被删除，这些依赖可能仍在 `node_modules` 中。
- 若追求精简，可分析后移除无用依赖
- 但若计划后续恢复某些模块，建议暂时保留依赖，避免重新安装时的版本冲突

## 5. 常见问题（FAQ）

### Q1：登录后侧边栏出现空白菜单项？
**原因**：后端返回了已删除模块的菜单，但前端找不到对应的 `.vue` 组件文件。  
**解决**：同步清理后端 `sys_menu` 中对应的菜单记录。

### Q2：新增页面后，菜单点击报错 "Cannot find module"？
**原因**：后端菜单的 `component` 路径与 `src/views/` 下的实际文件路径不一致。  
**解决**：检查 `component` 值是否与 `src/views/` 下的相对路径匹配（不含 `.vue` 后缀）。

### Q3：能否直接使用原 RuoYi 的代码生成器？
**回答**：当前项目已删除代码生成器前端页面（`views/tool/gen/`），但后端代码生成 API 仍保留在 `ruoyi-generator` 模块中。建议需要代码生成时：
- 方案 A：从官方仓库恢复 `views/tool/gen/` 及 `src/api/tool/gen.js`
- 方案 B：在完整版 RuoYi 项目中生成代码后，再迁移到当前项目

### Q4：`src/api/menu.js` 和 `src/router/index.js` 中的 `menu.js` 有什么区别？
**回答**：
- `src/api/menu.js`：调用后端接口 `/getRouters`，获取当前用户的动态菜单数据
- `src/router/index.js`：引入的是路由配置对象，与 `src/api/menu.js` 是不同的文件

### Q5：前端页面存在编码问题？
**回答**：当前 `frontend-dev` 分支在页面 rebranding（若依 → 灵档）过程中存在编码问题，详见 git log `adb12c0`，正在排查修复中。
