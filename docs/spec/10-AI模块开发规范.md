## 2. 命名规范

### 2.1 Java命名规范

| 类型 | 命名规则 | 示例 |
|------|----------|------|
| 类名 | 大驼峰 | `KbKnowledgeBaseServiceImpl` |
| 接口名 | I前缀 + 大驼峰 | `IKbKnowledgeBaseService` |
| 方法名 | 小驼峰，动词开头 | `listKnowledge`, `addDocument` |
| 变量名 | 小驼峰 | `knowledgeBase`, `docId` |
| 常量名 | 全大写下划线 | `DEFAULT_CHUNK_SIZE`, `MAX_FILE_SIZE` |
| 包名 | 全小写 | `com.ruoyi.lingdoc.knowledge` |

### 2.2 数据库命名规范

| 类型 | 命名规则 | 示例 |
|------|----------|------|
| 表名 | 小写下划线，kb_前缀 | `kb_knowledge_base` |
| 字段名 | 小写下划线 | `kb_id`, `create_time` |
| 主键 | 表名_id | `kb_id`, `doc_id` |
| 外键 | 被引表主键名 | `kb_id`, `doc_id` |
| 索引名 | idx_字段名 | `idx_kb_id`, `idx_user_uuid` |


**Controller实现：**
```java
@Slf4j
@Validated
@RestController
@RequestMapping("/lingdoc/knowledge")
public class KbKnowledgeBaseController extends BaseController {

    @Autowired
    private IKbKnowledgeBaseService kbService;

    /**
     * 查询知识库列表
     */
    @GetMapping("/list")
    public TableDataInfo list(KbKnowledgeBaseDTO dto) {
        return kbService.listKbKnowledgeBase(dto);
    }

    /**
     * 获取知识库详情
     */
    @GetMapping(value = "/{kbId}")
    public AjaxResult getInfo(@PathVariable String kbId) {
        KbKnowledgeBaseVO vo = kbService.getKbKnowledgeBase(kbId);
        return AjaxResult.success(vo);
    }

    /**
     * 新增知识库
     */
    @Log(title = "知识库", businessType = BusinessType.INSERT)
    @PostMapping
    public AjaxResult add(@RequestBody KbKnowledgeBaseDTO dto) {
        return toAjax(kbService.addKbKnowledgeBase(dto));
    }

    /**
     * 修改知识库
     */
    @Log(title = "知识库", businessType = BusinessType.UPDATE)
    @PutMapping
    public AjaxResult edit(@RequestBody KbKnowledgeBaseDTO dto) {
        return toAjax(kbService.updateKbKnowledgeBase(dto));
    }

    /**
     * 删除知识库
     */
    @Log(title = "知识库", businessType = BusinessType.DELETE)
    @DeleteMapping("/{kbId}")
    public AjaxResult remove(@PathVariable String kbId) {
        return toAjax(kbService.delKbKnowledgeBase(kbId));
    }
}
```

### 3.2 Vue编码规范

**页面组件结构：**
```vue
<template>
  <div class="app-container">
    <!-- 搜索表单 -->
    <el-form :model="queryParams" ref="queryRef" :inline="true">
      <!-- 表单内容 -->
    </el-form>

    <!-- 操作按钮 -->
    <el-row :gutter="10" class="mb8">
      <!-- 按钮内容 -->
    </el-row>

    <!-- 数据表格 -->
    <el-table v-loading="loading" :data="list">
      <!-- 表格列 -->
    </el-table>

    <!-- 分页 -->
    <pagination v-show="total > 0" :total="total" v-model:page="queryParams.pageNum" v-model:limit="queryParams.pageSize" @pagination="getList" />

    <!-- 新增/编辑弹窗 -->
    <el-dialog :title="title" v-model="open">
      <!-- 表单内容 -->
    </el-dialog>
  </div>
</template>

<script setup name="Knowledge">
import { listKnowledge, addKnowledge, updateKnowledge, delKnowledge } from '@/api/lingdoc/knowledge'

// 响应式数据
const loading = ref(false)
const open = ref(false)
const list = ref([])
const total = ref(0)
const title = ref('')

const queryParams = reactive({
  pageNum: 1,
  pageSize: 10,
  kbName: undefined
})

const form = reactive({
  kbId: undefined,
  kbName: '',
  kbDescription: ''
})

const rules = {
  kbName: [{ required: true, message: '知识库名称不能为空', trigger: 'blur' }]
}

// 方法
async function getList() {
  loading.value = true
  try {
    const res = await listKnowledge(queryParams)
    list.value = res.rows
    total.value = res.total
  } finally {
    loading.value = false
  }
}

function handleAdd() {
  reset()
  open.value = true
  title.value = '添加知识库'
}

function handleUpdate(row) {
  reset()
  Object.assign(form, row)
  open.value = true
  title.value = '修改知识库'
}

async function submitForm() {
  const valid = await proxy.$refs.formRef.validate()
  if (!valid) return
  
  if (form.kbId) {
    await updateKnowledge(form)
    proxy.$modal.msgSuccess('修改成功')
  } else {
    await addKnowledge(form)
    proxy.$modal.msgSuccess('新增成功')
  }
  open.value = false
  getList()
}

async function handleDelete(row) {
  await proxy.$modal.confirm('确认删除该知识库？')
  await delKnowledge(row.kbId)
  proxy.$modal.msgSuccess('删除成功')
  getList()
}

function reset() {
  Object.assign(form, {
    kbId: undefined,
    kbName: '',
    kbDescription: ''
  })
}

// 生命周期
onMounted(() => {
  getList()
})
</script>
```

## 5. 异常处理规范

### 5.1 后端异常

```java
// Service层抛出业务异常
public int addKbKnowledgeBase(KbKnowledgeBaseDTO dto) {
    // 参数校验
    if (StringUtils.isEmpty(dto.getKbName())) {
        throw new ServiceException("知识库名称不能为空");
    }
    
    // 重复性检查
    int count = baseMapper.countByName(userId, dto.getKbName());
    if (count > 0) {
        throw new ServiceException("知识库名称已存在");
    }
    
    try {
        return baseMapper.insert(entity);
    } catch (Exception e) {
        log.error("新增知识库失败", e);
        throw new ServiceException("系统繁忙，请稍后重试");
    }
}
```

### 5.2 前端异常

```javascript
async function submitForm() {
```

## 6. 日志规范

### 6.1 日志级别

| 级别 | 使用场景 | 示例 |
|------|----------|------|
| DEBUG | 调试信息，开发环境 | `log.debug("查询参数: {}", params)` |
| INFO | 业务流程记录 | `log.info("用户{}创建知识库{}", userId, kbId)` |
| WARN | 警告，非致命错误 | `log.warn("文件上传失败: {}", fileName)` |
| ERROR | 错误，需要处理 | `log.error("数据库操作失败", e)` |

### 6.2 日志内容

```java
// 入口日志
```
