# AI模块架构设计规范

> 本文档定义LingDoc AI模块的整体架构设计，严格遵循主仓库技术栈和编码规范。

---

## 1. 模块概述

### 1.1 功能范围

AI模块提供以下核心能力：

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 知识库管理 | 知识库的CRUD、配置参数管理 | P0 |
| 文档处理 | 文件上传、文本提取、智能分块 | P0 |
| 向量化存储 | Embedding生成、向量存储、索引管理 | P0 |
| RAG检索 | 语义检索、混合排序、上下文组装 | P0 |
| AI对话 | 多轮对话、流式响应、会话管理 | P0 |
| 模型管理 | 模型配置、密钥管理、调用统计 | P1 |

### 1.2 技术栈（与主仓库一致）

| 层级 | 技术 | 版本 | 说明 |
|------|------|------|------|
| 前端框架 | Vue | 3.5.26 | 组合式API + `<script setup>` |
| 构建工具 | Vite | 6.4.1 | 开发服务器、代理配置 |
| UI组件库 | Element Plus | 2.13.1 | 表格、表单、弹窗 |
| 后端框架 | Spring Boot | 4.0.3 | 主框架 |
| ORM | MyBatis Plus | 4.0.1 | 简化CRUD操作 |
| 数据库 | MySQL | 8.0+ | utf8mb4字符集 |
| 缓存 | Redis | 5.0+ | Token、会话缓存 |
| 向量存储 | MySQL | 8.0+ | BLOB存储向量（第一阶段） |

## 2. 架构设计

### 2.1 模块划分

```
com.ruoyi.lingdoc
├── ai/                          # AI核心模块
│   ├── controller/              # AI对话、模型管理Controller
│   ├── domain/                  # 实体类
│   │   ├── entity/              # 数据库实体
│   │   ├── dto/                 # 数据传输对象
│   │   └── vo/                  # 视图对象
│   ├── service/                 # 业务逻辑层
│   │   └── impl/                # 实现类
│   └── mapper/                  # 数据访问层
│
├── knowledge/                   # 知识库模块
│   ├── controller/              # 知识库、文档Controller
│   ├── domain/
│   ├── service/
│   └── mapper/
│
└── common/                      # AI模块公共组件
    ├── config/                  # 配置类
    ├── constant/                # 常量定义
    ├── util/                    # 工具类
    └── handler/                 # 处理器
```

### 2.2 核心流程

#### 2.2.1 文档处理流程

```
用户上传文件
    ↓
文件校验（类型、大小）
    ↓
保存到本地/云存储
    ↓
文本提取（Apache Tika）
    ↓
智能分块（段落+固定长度+重叠）
    ↓
生成Embedding
    ↓
存储向量到数据库
    ↓
更新知识库统计
```

#### 2.2.2 RAG对话流程

```
用户发送消息
    ↓
向量检索（Top-K相似分块）
    ↓
上下文组装（System Prompt + 检索结果）
    ↓
调用LLM API
    ↓
流式响应返回
    ↓
保存对话记录
```

## 3. 接口设计原则

### 3.1 URL规范

```
/lingdoc/{模块}/{资源}/{操作}

示例：
- GET    /lingdoc/knowledge/list          # 查询知识库列表
- POST   /lingdoc/knowledge               # 创建知识库
- GET    /lingdoc/knowledge/{id}          # 获取知识库详情
- PUT    /lingdoc/knowledge               # 更新知识库
- DELETE /lingdoc/knowledge/{id}          # 删除知识库

- POST   /lingdoc/doc/upload/{kbId}       # 上传文档
- GET    /lingdoc/doc/list/{kbId}         # 查询文档列表
- DELETE /lingdoc/doc/{docId}             # 删除文档

- POST   /lingdoc/chat/stream             # 流式对话
- GET    /lingdoc/chat/session/list       # 查询会话列表
```

### 3.2 权限前缀

```
lingdoc:{模块}:{操作}

示例：
- lingdoc:knowledge:list      # 查看知识库列表
- lingdoc:knowledge:add       # 新增知识库
- lingdoc:knowledge:edit      # 编辑知识库
- lingdoc:knowledge:remove    # 删除知识库

- lingdoc:doc:list            # 查看文档列表
- lingdoc:doc:add             # 上传文档
- lingdoc:doc:remove          # 删除文档

- lingdoc:chat:list           # 查看对话记录
- lingdoc:chat:add            # 发起对话
```

### 3.3 命名规范

**Java后端：**
- 类名：大驼峰，如 `KbKnowledgeBaseServiceImpl`
- 方法名：小驼峰，动词开头，如 `listKnowledgeBase`, `addDocument`
- 接口名：I前缀，如 `IKbKnowledgeBaseService`
- 常量：全大写下划线，如 `DEFAULT_CHUNK_SIZE`

**JavaScript前端：**
- API函数：动词-名词，如 `listKnowledge`, `addKnowledge`
- 变量：语义化，如 `knowledgeList`, `queryParams`
- 组件名：大驼峰，如 `KnowledgeIndex.vue`

## 4. 数据库设计规范

### 4.1 命名规范

| 类型 | 命名规则 | 示例 |
|------|----------|------|
| 表名 | `kb_`前缀 | `kb_knowledge_base` |
| 主键 | `表名_id` | `kb_id`, `doc_id` |
| 外键 | 被引表主键名 | `kb_id`, `doc_id` |
| 状态字段 | `status` | `0=正常 1=停用` |
| 时间字段 | `create_time`, `update_time` | 自动生成 |

### 4.2 字段类型规范

| 数据类型 | Java类型 | MySQL类型 | 说明 |
|----------|----------|-----------|------|
| ID | String | varchar(64) | UUID |
| 名称 | String | varchar(128) | 限制长度 |
| 描述 | String | varchar(512) | 长文本 |
| 内容 | String | longtext | 大文本 |
| 状态 | String | char(1) | 0/1 |
| 数字 | Integer | int | 计数、序号 |
| 时间 | LocalDateTime | datetime | 创建/更新时间 |
| 向量 | byte[] | blob | 二进制存储 |
| JSON | String | json | MySQL 5.7+ |

### 4.3 索引规范

- 主键：必须
- 外键：必须
- 查询条件字段：视情况添加
- 全文检索：`FULLTEXT`索引（仅MyISAM/InnoDB 5.6+）

## 5. 代码规范

### 5.1 后端代码规范

**Controller层：**
```java
@RestController
@RequestMapping("/lingdoc/knowledge")
public class KbKnowledgeBaseController extends BaseController {

    @Autowired
    private IKbKnowledgeBaseService kbService;

    @GetMapping("/list")
    public TableDataInfo list(KbKnowledgeBaseDTO dto) {
        return kbService.listKbKnowledgeBase(dto);
    }

    @PostMapping
    public AjaxResult add(@RequestBody KbKnowledgeBaseDTO dto) {
        return toAjax(kbService.addKbKnowledgeBase(dto));
    }
}
```

**Service层：**
```java
@Service
public class KbKnowledgeBaseServiceImpl 
    extends ServiceImpl<KbKnowledgeBaseMapper, KbKnowledgeBase> 
    implements IKbKnowledgeBaseService {

    @Override
    public TableDataInfo listKbKnowledgeBase(KbKnowledgeBaseDTO dto) {
        // 权限校验
        LoginUser loginUser = SecurityUtils.getLoginUser();
        
        // 构建查询条件
        LambdaQueryWrapper<KbKnowledgeBase> wrapper = new LambdaQueryWrapper<>();
        wrapper.eq(KbKnowledgeBase::getUserUuid, loginUser.getUserId().toString())
               .eq(KbKnowledgeBase::getStatus, "0")
               .orderByDesc(KbKnowledgeBase::getCreateTime);

        // 分页查询
        Page<KbKnowledgeBase> page = new Page<>(dto.getPageNum(), dto.getPageSize());
        page(page, wrapper);

        // 转换VO
        List<KbKnowledgeBaseVO> voList = page.getRecords().stream()
                .map(this::convertToVO)
                .collect(Collectors.toList());

        TableDataInfo dataInfo = new TableDataInfo();
        dataInfo.setRows(voList);
        dataInfo.setTotal(page.getTotal());
        return dataInfo;
    }
}
```

### 5.2 前端代码规范

**API层：**
```javascript
// src/api/lingdoc/knowledge.js
import request from '@/utils/request'

export function listKnowledge(query) {
  return request({
    url: '/lingdoc/knowledge/list',
    method: 'get',
    params: query
  })
}

export function addKnowledge(data) {
  return request({
    url: '/lingdoc/knowledge',
    method: 'post',
    data: data
  })
}
```

**Vue页面：**
```vue
<script setup name="Knowledge">
import { listKnowledge, addKnowledge } from '@/api/lingdoc/knowledge'

const knowledgeList = ref([])
const queryParams = reactive({
  pageNum: 1,
  pageSize: 10,
  kbName: ''
})

async function getList() {
  loading.value = true
  try {
    const res = await listKnowledge(queryParams)
    knowledgeList.value = res.rows
    total.value = res.total
  } catch (error) {
    console.error('获取列表失败:', error)
  } finally {
    loading.value = false
  }
}

onMounted(() => {
  getList()
})
</script>
```

## 6. 异常处理规范

### 6.1 后端异常

- 使用RuoYi框架统一异常处理
- Service层抛出业务异常：`throw new ServiceException("知识库不存在")`
- Controller层使用 `@Log` 注解记录操作日志

### 6.2 前端异常

- API调用使用 `try-catch` 包装
- 错误提示使用 `ElMessage.error()`
- 加载状态使用 `loading` 变量控制

## 7. 安全规范

### 7.1 权限控制

- 所有接口必须添加权限注解：`@PreAuthorize("@ss.hasPermi('lingdoc:knowledge:list')")`
- 数据权限：用户只能访问自己的知识库
- 敏感操作（删除）需要二次确认

### 7.2 数据安全

- API密钥不存储在数据库，使用环境变量
- 向量数据使用BLOB存储，加密敏感字段
- 文件上传限制类型和大小
