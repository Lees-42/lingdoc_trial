# 前后端代理与跨域配置

本文档解读开发环境下前端如何与后端通信，重点分析 Vite 的代理配置原理和跨域解决方案。

---

## 1. 核心文件

```
vite.config.js
```

---

## 2. 为什么需要代理

开发环境下：
- **前端**运行在 `http://localhost:80`
- **后端**运行在 `http://localhost:8080`

由于**同源策略**（Same-Origin Policy），浏览器会阻止前端直接向后端发送跨域请求。Vite 开发服务器提供的**反向代理**功能解决了这个问题。

---

## 3. Vite 代理配置详解

```javascript
const baseUrl = 'http://localhost:8080' // 后端接口地址

export default defineConfig(({ mode, command }) => {
  return {
    server: {
      port: 80,
      proxy: {
        // 开发 API 代理
        '/dev-api': {
          target: baseUrl,           // 目标后端地址
          changeOrigin: true,        // 修改请求头中的 origin
          rewrite: (p) => p.replace(/^\/dev-api/, '')
                                     // 去掉 /dev-api 前缀
        },
        // Swagger API 文档代理
        '^/v3/api-docs/(.*)': {
          target: baseUrl,
          changeOrigin: true
        }
      }
    }
  }
})
```

### 3.1 配置项说明

| 配置项 | 作用 |
|--------|------|
| `target` | 后端服务器地址 |
| `changeOrigin: true` | 将请求头中的 `Host` 改为后端地址，避免后端虚拟主机解析问题 |
| `rewrite` | URL 重写规则，将 `/dev-api/system/user/list` 改为 `/system/user/list` |

### 3.2 请求转发示例

```
前端代码：
  axios.get('/dev-api/system/user/profile')

浏览器实际发送：
  GET http://localhost/dev-api/system/user/profile

Vite 代理后转发给后端：
  GET http://localhost:8080/system/user/profile

后端接收：
  GET /system/user/profile（Host: localhost:8080）
```

---

## 4. 前端环境变量

```
.env.development
```

```
VITE_APP_BASE_API = '/dev-api'
```

Axios 实例创建时读取该变量：

```javascript
const service = axios.create({
  baseURL: import.meta.env.VITE_APP_BASE_API  // '/dev-api'
})
```

这意味着前端所有 API 请求都会自动加上 `/dev-api` 前缀，从而被 Vite 代理拦截并转发到后端。

---

## 5. 生产环境如何处理

开发环境使用 Vite 代理，但**生产环境不会启动 Vite 开发服务器**，因此代理不会生效。

### 5.1 生产环境部署方式

```
方式一：Nginx 反向代理（推荐）
  浏览器 → Nginx（80端口）
    ├── 静态资源请求 → 返回 dist/ 文件
    └── API 请求（/dev-api）→ proxy_pass 到后端 8080

方式二：Spring Boot 托管前端
  将 dist/ 文件放入 ruoyi-admin/src/main/resources/static/
  访问路径无需跨域（同域同端口）

方式三：Electron 桌面应用
  前端作为本地文件加载，API 请求直接访问本地后端
```

### 5.2 Nginx 配置示例

```nginx
server {
    listen 80;
    server_name localhost;

    location / {
        root /path/to/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    location /dev-api/ {
        proxy_pass http://localhost:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 6. Swagger 文档代理

```javascript
'^/v3/api-docs/(.*)': {
  target: baseUrl,
  changeOrigin: true
}
```

开发时访问 `http://localhost/v3/api-docs` 可直接查看后端 OpenAPI 文档的 JSON 定义，无需单独打开 8080 端口。

---

## 7. 代理链路总结

```
【开发环境】
浏览器 http://localhost
    ↓
Vite Dev Server (port 80)
    ├── /dev-api/xxx → proxy → http://localhost:8080/xxx
    └── /static/xxx → 返回本地构建产物

【生产环境（Nginx）】
浏览器 http://localhost
    ↓
Nginx (port 80)
    ├── /dev-api/xxx → proxy_pass → http://localhost:8080/xxx
    └── /xxx.html → 返回 dist/ 目录静态文件
```
