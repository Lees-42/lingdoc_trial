# 项目启动与运行总览

本文档描述灵档（LingDoc）项目的前后端启动方式、运行时端口，以及请求从前端到后端的完整生命周期。

---

## 1. 前端启动流程

### 1.1 启动命令

```bash
npm run dev
```

### 1.2 内部执行链路

```
npm run dev
    ↓
package.json scripts.dev → "vite"
    ↓
Vite 读取 vite.config.js
    ↓
加载 Vite 插件（createVitePlugins）
    ↓
启动开发服务器（默认端口 80）
    ↓
自动打开浏览器访问 http://localhost
```

### 1.3 关键配置

- **入口 HTML**：`index.html`
- **入口 JS**：`src/main.js`
- **开发服务器端口**：`80`（`vite.config.js` 中 `server.port`）
- **环境变量**：`.env.development` 中定义 `VITE_APP_BASE_API = '/dev-api'`

---

## 2. 后端启动流程

### 2.1 启动方式

**方式一：IDE 运行**
- 直接运行 `ruoyi-server/ruoyi-admin/src/main/java/com/ruoyi/RuoYiApplication.java` 的 `main()` 方法

**方式二：命令行运行 jar**
```bash
cd ruoyi-server
mvn clean package -DskipTests
java -jar ruoyi-admin/target/ruoyi-admin.jar
```

### 2.2 内部执行链路

```
RuoYiApplication.main()
    ↓
SpringApplication.run(RuoYiApplication.class, args)
    ↓
@SpringBootApplication 自动配置
    ↓
排除 DataSourceAutoConfiguration（由 Druid 手动配置）
    ↓
加载 application.yml + application-druid.yml
    ↓
初始化 Spring Security 过滤器链
    ↓
初始化 MyBatis + Druid 数据源
    ↓
内嵌 Tomcat 启动（端口 8080）
    ↓
控制台输出启动成功日志
```

### 2.3 关键配置

- **主配置**：`ruoyi-server/ruoyi-admin/src/main/resources/application.yml`
- **数据源配置**：`ruoyi-server/ruoyi-admin/src/main/resources/application-druid.yml`
- **服务器端口**：`8080`
- **数据库**：MySQL 8.0+
- **缓存**：Redis 5.0+

## 3. 前后端交互完整链路

以下是一条典型请求（如用户点击"个人中心"）从前端到后端的完整生命周期：

```
浏览器访问 http://localhost
        ↓
前端 Vite Dev Server（端口 80）
        ↓
Vue Router 匹配路由 → 导航守卫（permission.js）
        ↓
检查 Cookie 中的 Token
        ↓
Axios 发送请求（如 GET /dev-api/system/user/profile）
        ↓
request.js 拦截器：添加 Authorization: Bearer <token>
        ↓
vite.config.js 代理 /dev-api → http://localhost:8080
        ↓
后端 ruoyi-admin（Tomcat / 端口 8080）
        ↓
Spring Security 过滤器链
        ↓
JwtAuthenticationTokenFilter 解析并验证 JWT Token
        ↓
SecurityContextHolder 设置当前登录用户
        ↓
Controller（如 SysProfileController）
        ↓
Service / Mapper（ruoyi-system）
        ↓
MySQL / Redis 查询
        ↓
统一响应体 AjaxResult（ruoyi-framework 封装）
        ↓
JSON 响应返回前端
        ↓
前端 request.js 响应拦截器处理 code / msg
        ↓
Vue 组件更新 Pinia Store 或本地状态
        ↓
页面渲染更新
```

## 4. 运行环境依赖

| 服务 | 版本要求 | 说明 |
|------|---------|------|
| Node.js | 18+ | 前端运行环境 |
| JDK | 17+ | 后端运行环境 |
| MySQL | 8.0+ | 业务数据持久化 |
| Redis | 5.0+ | Token、字典、会话缓存 |
| Maven | 3.8+ | 后端构建工具 |
