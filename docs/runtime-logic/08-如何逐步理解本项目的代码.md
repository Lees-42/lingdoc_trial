# 如何逐步理解本项目的代码

本文档提供一套可执行的步骤指南，帮助你由浅入深地理解灵档（LingDoc）项目的运行逻辑。

---

## Step 1：先看启动入口（5分钟）

**目标**：了解前后端各从哪个文件开始运行。

- [ ] 打开 `src/main.js`，浏览一遍导入顺序和 `app.mount('#app')`
- [ ] 打开 `ruoyi-server/ruoyi-admin/src/main/java/com/ruoyi/RuoYiApplication.java`，理解 `SpringApplication.run()`
- [ ] 记录：前后端各自的入口文件、开发端口、启动命令

**产出**：在脑中建立"前端从 main.js 开始，后端从 RuoYiApplication 开始"的基本认知。

---

## Step 2：跟一次完整登录流程（15分钟）

**目标**：理解用户登录时前后端如何交互。

### 前端侧跟踪

- [ ] 打开 `src/views/login.vue`，找到登录按钮的点击事件
- [ ] 跟踪到 `src/api/login.js` 的 `login()` 函数
- [ ] 查看 `src/utils/request.js`，确认请求如何附加 Token
- [ ] 查看 `src/utils/auth.js`，确认 Token 如何存入 Cookie

### 后端侧跟踪

- [ ] 打开 `ruoyi-admin/.../web/controller/common/SysLoginController.java`
- [ ] 找到 `login()` 方法，跟踪到 `SysLoginService`
- [ ] 查看 `TokenService.createToken()` 了解 JWT 生成逻辑
- [ ] 用 Redis 客户端（或命令行）查看登录后是否生成了 `login_tokens:*` 键

**关键问题**：
- Token 存在哪里？（Cookie / Redis）
- Token 长什么样？（JWT 三段式结构）
- 登录成功后前端做了什么？（跳转首页、获取用户信息）

---

## Step 3：跟一次页面跳转（15分钟）

**目标**：理解从登录后到看到首页，路由系统做了什么。

- [ ] 打开 `src/permission.js`，逐行阅读 `router.beforeEach`
- [ ] 在浏览器控制台输入 `router.getRoutes()`，观察动态路由是否已注册
- [ ] 打开 `src/store/modules/permission.js`，理解 `generateRoutes()`
- [ ] 查看 `src/store/modules/user.js` 的 `getInfo()` 方法
- [ ] 查看后端 `SysMenuController.getRouters()` 返回的数据格式

**关键问题**：
- 刷新页面后，为什么 roles.length === 0？
- 侧边栏菜单从哪里来？（后端 `sys_menu` 表）
- `router.addRoute()` 是在什么时候调用的？

---

## Step 4：跟一次数据请求（15分钟）

**目标**：理解从点击按钮到数据返回的完整链路。

**以"个人中心"为例**：

- [ ] 前端：`src/views/system/user/profile/index.vue`
- [ ] 前端 API：`src/api/system/user.js` 的 `getUserProfile()`
- [ ] 请求拦截：`src/utils/request.js` 附加 `Authorization: Bearer xxx`
- [ ] Vite 代理：`vite.config.js` 将 `/dev-api` 转发到 `localhost:8080`
- [ ] 后端控制器：`SysProfileController.getProfile()`
- [ ] 后端服务：`ruoyi-system/.../service/impl/SysUserServiceImpl`
- [ ] 数据层：`SysUserMapper.selectUserById()` → `SysUserMapper.xml`
- [ ] 数据库：MySQL `sys_user` 表
- [ ] 响应返回：经过 `ruoyi-framework` 的 `AjaxResult` 封装
- [ ] 响应拦截：`request.js` 解析 code === 200，返回数据
- [ ] 页面更新：Vue 组件更新状态，渲染页面

**调试技巧**：
- 浏览器 F12 → Network 标签，观察请求头、响应体
- 后端在 Controller 方法处打断点，逐步跟踪

---

## Step 5：挑一个业务模块纵向贯通（30分钟）

**目标**：从页面到数据库，完整走通一个业务功能。

**推荐模块**：参数配置（`views/system/config`）

因为它结构简单，但包含完整的增删改查流程：

```
前端页面
    ↓
前端 API（src/api/system/config.js）
    ↓
Axios 请求
    ↓
后端 Controller（SysConfigController）
    ↓
后端 Service（ISysConfigService / SysConfigServiceImpl）
    ↓
后端 Mapper（SysConfigMapper）
    ↓
MyBatis XML（SysConfigMapper.xml）
    ↓
数据库表（sys_config）
```

- [ ] 阅读 `views/system/config/index.vue`，理解表格渲染和分页
- [ ] 阅读 `src/api/system/config.js`，理解 API 函数命名规范
- [ ] 阅读后端三层代码（Controller → Service → Mapper）
- [ ] 对比 MyBatis XML 中的 SQL 与数据库表结构

---

## Step 6：用调试工具验证（20分钟）

### 浏览器端

- [ ] F12 → Network → 过滤 `Fetch/XHR`
- [ ] 观察登录请求的 Request Headers（Authorization）
- [ ] 观察响应的 Response 结构（code / msg / data）
- [ ] Vue DevTools（如有安装）查看 Pinia Store 状态

### 后端端

- [ ] 在 IDEA 中打开 Debug 模式运行 `RuoYiApplication`
- [ ] 在 `SysLoginController.login()` 打断点
- [ ] 在 `JwtAuthenticationTokenFilter.doFilterInternal()` 打断点
- [ ] 观察 `SecurityContextHolder` 中的认证信息

### Redis 端

```bash
# 连接 Redis
redis-cli

# 查看所有登录 Token
KEYS login_tokens:*

# 查看某个 Token 的内容
GET login_tokens:<uuid>
```

## 理解路线图总结

```
第 1 步：找到入口（main.js + RuoYiApplication）
    ↓
第 2 步：跟登录（Token 从哪来、存哪去）
    ↓
第 3 步：跟路由（页面怎么跳、菜单从哪来）
    ↓
第 4 步：跟请求（点按钮后发生了什么）
    ↓
第 5 步：纵向贯通（页面 → API → Controller → Service → Mapper → DB）
    ↓
第 6 步：工具验证（Network、Debug、Redis）
```

## 下一步建议

4. **扩展运行逻辑文档**：在 `docs/runtime-logic/` 中补充你发现的新链路（如文件上传、SSE 流式请求等）
