# 前端路由与导航守卫

本文档解读前端路由系统的运行逻辑，包括静态路由、动态路由、权限生成和导航守卫的完整执行链。

---

## 1. 核心文件

| 文件 | 职责 |
|------|------|
| `src/router/index.js` | 定义 `constantRoutes`（公共路由）和 `dynamicRoutes`（权限隐藏路由） |
| `src/permission.js` | 注册全局路由守卫 `beforeEach` / `afterEach` |
| `src/store/modules/permission.js` | 从后端获取菜单并生成可访问路由 |
| `src/store/modules/user.js` | 管理用户信息、Token、角色、权限 |

---

## 2. 路由类型说明

### 2.1 constantRoutes（公共路由）

定义在 `src/router/index.js` 中，**不依赖权限**，所有用户均可访问：

- `/login` — 登录页
- `/register` — 注册页
- `/index` — 首页
- `/401`、`/404` — 错误页
- `/user/profile` — 个人中心
- `/lock` — 锁屏页

### 2.2 dynamicRoutes（动态隐藏路由）

定义在 `src/router/index.js` 中，**需要特定权限**才能访问的前端隐藏页面：

- `/system/dict-data/index/:dictId` — 字典数据页（需要 `system:dict:list` 权限）

### 2.3 后端菜单路由

通过 `getRouters()` 接口从后端 `sys_menu` 表获取，用于：
- 渲染侧边栏菜单树
- 动态注册到 Vue Router

---

## 3. 导航守卫执行流程

当用户访问任意页面时，`src/permission.js` 中的 `router.beforeEach` 按以下顺序执行：

```
用户点击链接或输入 URL
    ↓
router.beforeEach 触发
    ↓
NProgress.start()（显示顶部进度条）
    ↓
检查 Cookie 中是否存在 Token（getToken()）
    ├── 有 Token
    │   ├── 目标页是 /login → 重定向到 /
    │   ├── 在白名单（/login、/register）→ 放行
    │   ├── 屏幕锁定且目标不是 /lock → 重定向到 /lock
    │   ├── 未获取过用户信息（roles.length === 0）
    │   │   ├── await useUserStore().getInfo()（调 /getInfo 接口）
    │   │   ├── await usePermissionStore().generateRoutes()（调 /getRouters 接口）
    │   │   ├── 遍历 accessRoutes，router.addRoute(route) 动态注册
    │   │   └── return { ...to, replace: true }（重新导航）
    │   └── 已获取用户信息 → 放行
    └── 无 Token
        ├── 目标页在白名单 → 放行
        └── 不在白名单 → 重定向到 /login?redirect=xxx
```

## 4. 动态路由生成详解

`src/store/modules/permission.js` 中的 `generateRoutes()`：

```javascript
generateRoutes() {
  return new Promise(resolve => {
    getRouters().then(res => {
      // 1. 菜单标题替换（智能检索 → 灵犀问答）
      renameMenuTitle(res.data, '智能检索', '灵犀问答')
      
      // 2. 将后端菜单字符串转换为 Vue 组件对象
      const sidebarRoutes = filterAsyncRouter(JSON.parse(JSON.stringify(res.data)))
      
      // 3. 处理扁平化路由（用于标签页和面包屑）
      const rewriteRoutes = filterAsyncRouter(..., false, true)
      
      // 4. 过滤并注册 dynamicRoutes
      const asyncRoutes = filterDynamicRoutes(dynamicRoutes)
      asyncRoutes.forEach(route => router.addRoute(route))
      
      // 5. 保存到 Pinia Store
      this.setRoutes(rewriteRoutes)
      this.setSidebarRouters(constantRoutes.concat(sidebarRoutes))
      
      resolve(rewriteRoutes)
    })
  })
}
```

### 4.1 组件映射规则

后端返回的菜单数据中的 `component` 字段需要映射为前端实际组件：

| 后端 component 值 | 实际映射 |
|------------------|---------|
| `Layout` | `src/layout/index.vue`（带侧边栏的布局） |
| `ParentView` | `src/components/ParentView.vue`（嵌套路由容器） |
| `InnerLink` | `src/layout/components/InnerLink.vue`（内嵌 iframe） |
| 其他（如 `system/user/index`） | `import('@/views/system/user/index.vue')`（懒加载） |

### 4.2 组件懒加载机制

```javascript
const modules = import.meta.glob('./../../views/**/*.vue')

export const loadView = (view) => {
  for (const path in modules) {
    const dir = path.split('views/')[1].split('.vue')[0]
    if (dir === view) {
      return () => modules[path]()
    }
  }
}
```

使用 `import.meta.glob` 批量匹配 `src/views/` 下的所有 `.vue` 文件，实现按需懒加载。

## 5. 路由守卫状态流转示例

**场景：已登录用户刷新浏览器，访问 `/system/user/profile`**

```
刷新页面 → beforeEach 触发
    ↓
getToken() 返回有效 Token
    ↓
目标不是 /login
    ↓
useUserStore().roles.length === 0（页面刷新后状态丢失）
    ↓
await useUserStore().getInfo()
    调 GET /dev-api/getInfo
    后端解析 JWT → 查询用户、角色、权限
    返回 { user, roles, permissions }
    Pinia 中存储用户信息
    ↓
await usePermissionStore().generateRoutes()
    调 GET /dev-api/getRouters
    后端根据用户权限查询 sys_menu
    返回菜单树 JSON
    前端映射为 Vue Router 路由对象
    router.addRoute() 逐个注册
    ↓
return { ...to, replace: true }
    重新导航到 /system/user/profile
    此时路由已注册，正常匹配
    ↓
router.afterEach → NProgress.done()
```
