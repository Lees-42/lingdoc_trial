# 表格填写助手 —— 设计逻辑文档

**文档编号**: LingDoc-FAST-009  
**版本**: v1.0  
**编制日期**: 2026-04-21  
**状态**: 定稿  

---

## 1. 文档定位

本文档定义「表格填写助手」模块在 AI 返回 JSON 数据后的**核心处理逻辑**，覆盖以下六个环节：

1. 空白表格由用户上传（复用 FAST-003 接口）
2. 上传后调用 AI 接口，对接另一位开发者实现的 AI 逻辑
3. AI 返回 JSON 数据，驱动后续所有流程
4. 返回 JSON 后**页面布局保持不变**，仅更新右侧预览内容
5. 添加「保存到 Vault」功能，将填写结果归档到 LingDoc 本地文件系统
6. 添加「下载结果」功能，将填写后的文档下载到用户本地磁盘

> **衔接关系**：本文档与 FAST-001（产品需求）、FAST-002（数据库设计）、FAST-003（前后端接口规范）、FAST-004（AI 开发者接口契约）互补。FAST-003/004 定义了「有什么接口」，本文档定义了「数据如何在接口之间流动、如何渲染、如何归档」。

### 2.2 状态流转（扩展）

在 FAST-001 定义的五态模型基础上，本文档补充「已保存到 Vault」的判定逻辑：

```
[待上传:0] → [识别中:1] → [待确认:2] → [已生成:3] → [完成]
                              │              │
                              │              ├─ 是否 vault_file_id != NULL ?
                              │              │     ├─ 是 → [已归档]（隐式状态）
                              │              │     └─ 否 → [未归档]（隐式状态）
                              │              │
                              └──────────────┘
                                   重新生成
```

> **设计决策**：不新增独立状态码，而是通过 `lingdoc_form_task.filled_file_id` 是否关联到 `lingdoc_file_index` 记录来判断是否已归档到 Vault。这样保持状态机的简洁性，同时满足业务判断需求。

## 3. AI JSON 返回格式与处理逻辑

### 3.1 第一次 AI 调用：字段识别（extractFields）

**调用方**：`LingdocFormController.upload()` → `ILingdocFormTaskService.extractFields(taskId)`  
**实现方**：另一位 AI 开发者  
**返回格式**：JSON（见 FAST-004 §3.1），核心为 `fields` 数组与 `references` 数组。

**后端处理逻辑**：

```java
// LingdocFormTaskServiceImpl.extractFields(taskId) 中的处理流程

public void extractFields(String taskId) {
    // 1. 根据 taskId 读取原始文件路径
    LingdocFormTask task = formTaskMapper.selectLingdocFormTaskById(taskId);
    String filePath = RuoYiConfig.getUploadPath() + task.getOriginalFileUrl();

    // 2. 调用 AI 开发者实现的识别逻辑（或 Feign 调用 AI 服务）
    //    入参：taskId + filePath + fileType + referenceDocIds
    //    出参：AiExtractResult JSON
    AiExtractResult result = aiFormService.extract(filePath, task.getOriginalFileName());

    // 3. 将 AI 返回的 JSON 中的 fields 写入 lingdoc_form_field
    for (AiField field : result.getFields()) {
        LingdocFormField formField = new LingdocFormField();
        formField.setFieldId(UUID.fastUUID().toString());
        formField.setTaskId(taskId);
        formField.setFieldName(field.getFieldName());      // e.g. "姓名"
        formField.setFieldType(field.getFieldType());      // e.g. "text"
        formField.setFieldLabel(field.getFieldLabel());    // e.g. "姓名（中文）"
        formField.setAiValue(field.getSuggestedValue());   // e.g. "张三"
        formField.setConfidence(field.getConfidence());    // e.g. 0.92
        formField.setSourceDocId(field.getSourceDocId());
        formField.setSourceDocName(field.getSourceDocName());
        formField.setSortOrder(field.getSortOrder());
        formField.setIsConfirmed("0");
        formFieldMapper.insertLingdocFormField(formField);
    }

    // 4. 将 references 写入 lingdoc_form_reference
    for (AiReference ref : result.getReferences()) {
        LingdocFormReference formRef = new LingdocFormReference();
        formRef.setRefId(UUID.fastUUID().toString());
        formRef.setTaskId(taskId);
        formRef.setDocId(ref.getDocId());
        formRef.setDocName(ref.getDocName());
        formRef.setDocPath(ref.getDocPath());
        formRef.setDocType(ref.getDocType());
        formRef.setRelevance(ref.getRelevance());
        formRef.setIsSelected("1");  // 默认选中
        formReferenceMapper.insertLingdocFormReference(formRef);
    }

    // 5. 更新任务状态为「待确认:2」
    task.setStatus("2");
    task.setFieldCount(result.getFields().size());
    task.setTokenCost(result.getTokenCost());
    formTaskMapper.updateLingdocFormTask(task);
}
```

### 3.2 第二次 AI 调用：文档生成（generateDocument）

**调用方**：`LingdocFormController.generate()` → `ILingdocFormTaskService.generateDocument(taskId)`  
**实现方**：另一位 AI 开发者  
**返回格式**：JSON，包含 `filledValues`（字段值映射表）。

> **关键设计**：AI 开发者可以选择两种实现方式之一返回结果：
> - **方式 A**：AI 直接生成填写后的文件（PDF/Word/HTML），返回文件路径。
> - **方式 B**：AI 返回字段值映射 JSON，由后端根据原始模板文件 + JSON 数据渲染生成预览文件。
>
> 本文档推荐 **方式 B**，理由：
> 1. 前端页面布局不变，右侧 iframe 始终加载一个 URL
> 2. 后端可控渲染过程，便于调试和维护
> 3. 与 HTML 模板（如 `国家奖学金申请表.html`）的 `.blank` → `.filled` 替换逻辑天然契合

**后端渲染逻辑（HTML 模板为例）**：

```java
public String generateDocument(String taskId) {
    // 1. 读取原始文件
    LingdocFormTask task = formTaskMapper.selectLingdocFormTaskById(taskId);
    String originalPath = RuoYiConfig.getUploadPath() + task.getOriginalFileUrl();

    // 2. 调用 AI 接口，获取 filledValues JSON
    AiGenerateResult result = aiFormService.generate(taskId, originalPath, confirmedFields);
    Map<String, String> filledValues = result.getFilledValues();

    // 3. 读取原始 HTML 内容
    String htmlContent = FileUtils.readString(originalPath);

    // 4. 替换逻辑：将 .blank 占位符替换为 .filled 实际值
    // 规则：根据 <th>字段名</th> 定位同级的 <td class="blank">...
    // 使用 DOM 解析（Jsoup）比正则更可靠
    Document doc = Jsoup.parse(htmlContent);
    Elements thElements = doc.select("th");
    for (Element th : thElements) {
        String fieldName = th.text().trim();
        if (filledValues.containsKey(fieldName)) {
            // 找到同 tr 中 th 之后的第一个 td
            Element td = th.parent().select("td.blank").first();
            if (td != null) {
                td.removeClass("blank").addClass("filled");
                td.text(filledValues.get(fieldName));
            }
        }
    }

    // 5. 保存生成的临时预览文件
    String filledFileName = task.getOriginalFileName().replaceAll("\\.(?=[^\\.]+$)", "_已填写_" + DateUtils.dateTimeNow() + ".$0");
    String filledDir = RuoYiConfig.getUploadPath() + "/lingdoc/form/filled/" + DateUtils.datePath();
    File filledDirFile = new File(filledDir);
    if (!filledDirFile.exists()) filledDirFile.mkdirs();
    String filledPath = filledDir + "/" + filledFileName;
    FileUtils.writeString(filledPath, doc.html());

    // 6. 更新任务状态
    task.setStatus("3");
    task.setFilledFileUrl(filledPath.replace(RuoYiConfig.getUploadPath(), ""));
    task.setFilledFileName(filledFileName);
    task.setTokenCost(task.getTokenCost() + result.getTokenCost());
    formTaskMapper.updateLingdocFormTask(task);

    return filledPath;
}
```

> **PDF/Word 格式说明**：如果原始文件是 PDF 或 Word，渲染逻辑不同：
> - **PDF**：使用 PDFBox/iText 填充 AcroForm 字段，或先生成 HTML 再转 PDF。
> - **Word**：使用 Apache POI 操作 docx 的 Content Control / 书签，替换占位符。
> - **兜底策略**：如果格式不支持智能渲染，AI 服务直接返回填写后的文件流，后端透传保存。

## 4. 页面渲染逻辑（页面不变原则）

### 4.1 核心原则

> **「返回后的给用户展示的页面不变」**：无论 AI 返回的是字段识别 JSON 还是生成后的填充 JSON，前端页面的整体布局（左右分栏预览 + 中间字段编辑面板 + 底部参考文档）**不发生结构性变化**。变化仅体现在数据层面和右侧 iframe 的 URL 更新。

### 4.2 布局稳定性保证

前端 `src/views/lingdoc/form/index.vue` 中的 DOM 结构固定如下：

```vue
<div class="form-main">
  <!-- 顶部工具栏 -->
  <div class="form-toolbar">...</div>

  <!-- 左右分栏预览区 -->
  <div class="preview-area">
    <div class="preview-panel"> <!-- 左侧：原始文档 -->
      <iframe :src="originalFileUrl" />
    </div>
    <div class="preview-panel"> <!-- 右侧：AI 填写后 -->
      <iframe :src="filledFileUrl" />
    </div>
  </div>

  <!-- 字段编辑面板 -->
  <div class="field-editor-panel">
    <el-table :data="fields">...</el-table>
  </div>

  <!-- 参考文档面板 -->
  <div class="reference-panel">...</div>
</div>
```

**数据变化 vs 布局变化**：

| 阶段 | 数据变化 | 布局变化 |
|------|---------|---------|
| 上传后 | `currentTask` 赋值，`fields` 为空 | 无，显示空状态或 loading |
| AI 识别返回 | `fields` 数组填充，`references` 数组填充 | 无，字段表格自动渲染 |
| 用户确认字段 | `fields[].userValue` / `isConfirmed` 更新 | 无，表格单元格响应式更新 |
| AI 生成返回 | `filledFileUrl` 更新 | 无，右侧 iframe `src` 自动刷新 |
| 保存到 Vault | `currentTask.vaultFileId` 标记 | 无，按钮状态或 toast 提示 |

### 4.3 右侧 iframe 刷新机制

当 `filledFileUrl` 从 `null` 变为有值时，Vue 的响应式绑定会自动触发右侧 iframe 重新加载：

```javascript
// 前端 loadTaskDetail 中
if (currentTask.value.filledFileUrl) {
  // 拼接完整 URL：base_api + 相对路径
  filledFileUrl.value = import.meta.env.VITE_APP_BASE_API + currentTask.value.filledFileUrl
}
```

> **注意事项**：iframe 加载同域 HTML 文件时无跨域问题。如果生成的是 PDF/Word，iframe 会调用浏览器内置预览器，行为一致。

## 5. 保存到 Vault 设计

### 5.1 功能定义

用户点击「保存到 Vault」后，系统将当前已生成的填写后文件**归档到 LingDoc Vault 本地文件系统**，使其成为 Vault 中的正式文件，可被文件浏览器检索、被其他 AI 功能引用。

### 5.2 前端交互

**新增按钮**：在顶部工具栏「下载结果」旁增加「保存到 Vault」按钮。

```vue
<el-button type="info" :icon="FolderChecked" :disabled="!canSaveToVault" @click="handleSaveToVault">
  保存到 Vault
</el-button>
```

**计算属性**：

```javascript
const canSaveToVault = computed(() => {
  // 已生成 + 有 filledFileUrl + 未保存过
  return currentTask.value
    && currentTask.value.status === '3'
    && currentTask.value.filledFileUrl
    && !currentTask.value.vaultFileId  // 新增字段，标识是否已归档
})
```

### 5.4 后端处理逻辑

```java
@PreAuthorize("@ss.hasPermi('lingdoc:form:edit')")
@Log(title = "表格填写助手", businessType = BusinessType.INSERT)
@PostMapping("/saveToVault")
public AjaxResult saveToVault(@RequestBody LingdocFormTask taskParam) {
    String taskId = taskParam.getTaskId();
    LingdocFormTask task = formTaskService.selectLingdocFormTaskById(taskId);
    if (task == null) {
        return AjaxResult.error("任务不存在");
    }
    if (StringUtils.isEmpty(task.getFilledFileUrl())) {
        return AjaxResult.error("尚未生成填写后文档，无法保存");
    }

    // 1. 确定源文件（临时生成文件）
    String srcPath = RuoYiConfig.getUploadPath() + task.getFilledFileUrl();
    File srcFile = new File(srcPath);
    if (!srcFile.exists()) {
        return AjaxResult.error("生成文件已失效，请重新生成");
    }

    // 2. 确定 Vault 目标路径
    // Vault 根目录配置：RuoYiConfig.getProfile() + "/vault/" + userId + "/documents/"
    String vaultRoot = RuoYiConfig.getProfile() + "/vault/" + getUserId() + "/documents/表格助手/";
    File vaultDir = new File(vaultRoot);
    if (!vaultDir.exists()) vaultDir.mkdirs();

    String vaultFileName = task.getFilledFileName();
    String vaultAbsPath = vaultRoot + vaultFileName;

    // 3. 复制文件到 Vault（避免移动导致临时预览失效）
    FileUtils.copyFile(srcPath, vaultAbsPath);

    // 4. 计算 SHA256 checksum
    String checksum = FileUtils.calculateChecksum(vaultAbsPath, "SHA-256");

    // 5. 写入 lingdoc_file_index
    LingdocFileIndex fileIndex = new LingdocFileIndex();
    String fileId = UUID.fastUUID().toString();
    fileIndex.setFileId(fileId);
    fileIndex.setUserId(getUserId());
    fileIndex.setFileName(vaultFileName);
    fileIndex.setVaultPath("表格助手/" + vaultFileName);
    fileIndex.setAbsPath(vaultAbsPath);
    fileIndex.setFileType(getFileExtension(vaultFileName));  // html/pdf/docx 等
    fileIndex.setFileSize(srcFile.length());
    fileIndex.setChecksum(checksum);
    fileIndex.setSourceType("2");  // 2 = 表格助手生成
    fileIndex.setCreateTime(DateUtils.getNowDate());
    lingdocFileIndexMapper.insertLingdocFileIndex(fileIndex);

    // 6. 更新任务表的关联字段
    task.setFilledFileId(fileId);  // 关联到 Vault 文件
    formTaskService.updateLingdocFormTask(task);

    // 7. 异步触发文本解析（写入 lingdoc_file_ai_meta）
    // 使该文件可被 Vault 搜索和其他 AI 功能引用
    lingdocFileAiMetaService.initMeta(fileId);

    return AjaxResult.success("已保存到 Vault")
        .put("vaultFileId", fileId)
        .put("vaultFileName", vaultFileName)
        .put("vaultPath", fileIndex.getVaultPath());
}
```

### 5.5 数据库变更（最小化）

现有 `lingdoc_form_task` 表已包含 `filled_file_id` 字段（见 FAST-002 §2.1），但当前含义是「生成后文档文件ID」。本文档将其语义扩展为：

- 当 `filled_file_id` 指向一个**上传路径下的临时文件 ID** 时，文件未归档。
- 当 `filled_file_id` 指向 `lingdoc_file_index.file_id` 时，文件已归档到 Vault。

> **判定方法**：查询 `lingdoc_file_index` 是否存在 `file_id = filled_file_id`，若存在则已归档。

## 6. 下载功能设计

### 6.1 功能定位

下载功能已在 FAST-003 §2.6 和前端 `form/index.vue` 中定义和实现。本文档补充**文件来源优先级**和**与 Vault 保存的联动逻辑**。

### 6.2 文件来源优先级

后端 `LingdocFormController.download()` 按以下优先级确定文件来源：

| 优先级 | 来源 | 判定条件 | 说明 |
|--------|------|---------|------|
| 1 | Vault 归档文件 | `filled_file_id` 存在于 `lingdoc_file_index` | 用户已执行「保存到 Vault」，从 Vault 路径读取 |
| 2 | 临时生成文件 | `filled_file_url` 不为空 | 用户尚未保存到 Vault，从 `/upload/lingdoc/form/filled/` 读取 |
| 3 | 失败 | 以上都不满足 | 返回 404 |

```java
@GetMapping("/download/{taskId}")
public void download(@PathVariable("taskId") String taskId, HttpServletResponse response) {
    LingdocFormTask task = formTaskService.selectLingdocFormTaskById(taskId);
    if (task == null) { ... }

    String filePath;
    String downloadName;

    // 优先级 1：Vault 归档文件
    if (StringUtils.isNotEmpty(task.getFilledFileId())) {
        LingdocFileIndex fileIndex = lingdocFileIndexMapper.selectById(task.getFilledFileId());
        if (fileIndex != null) {
            filePath = fileIndex.getAbsPath();
            downloadName = fileIndex.getFileName();
        }
    }

    // 优先级 2：临时生成文件
    if (filePath == null && StringUtils.isNotEmpty(task.getFilledFileUrl())) {
        filePath = RuoYiConfig.getUploadPath() + task.getFilledFileUrl();
        downloadName = task.getFilledFileName();
    }

    // 响应文件流
    response.setContentType("application/octet-stream");
    FileUtils.setAttachmentResponseHeader(response, downloadName);
    FileUtils.writeBytes(filePath, response.getOutputStream());
}
```

### 6.3 前端下载逻辑

前端已实现 Blob 下载，无需修改：

```javascript
async function handleDownload() {
  const res = await downloadForm(currentTask.value.taskId)
  const blob = new Blob([res], { type: res.type || 'application/octet-stream' })
  const link = document.createElement('a')
  link.href = URL.createObjectURL(blob)
  link.download = currentTask.value.filledFileName || '已填写表格.html'

  URL.revokeObjectURL(link.href)
}
```
