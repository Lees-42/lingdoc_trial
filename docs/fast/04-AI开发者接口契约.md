# AI 开发者接口契约 —— 表格填写助手

**文档编号**: LingDoc-FAST-004  
**版本**: v1.0  
**编制日期**: 2026-04-19  
**状态**: 草案（待另一位 AI 开发者确认）  

---

## 1. 概述

本文档定义了**表格填写助手模块**与 **AI 服务模块**之间的协作规范。

**协作方式**：AI 识别逻辑由 AI 开发者在 Service 层实现（或作为独立服务通过 Feign 调用），后端开发者负责在 `LingdocFormTaskService` 中集成调用。  
**当前状态**：`LingdocFormController.upload()` 中已预留 TODO 注释，待接入真实 AI 识别逻辑。

---

## 2. 接口清单

| 序号 | 接口名称 | 实现位置 | 说明 |
|------|----------|---------|------|
| 1 | 识别表格字段 | `ILingdocFormTaskService.extractFields(taskId)` | 上传空白表格，解析并返回字段列表 |
| 2 | 生成填写后文档 | `ILingdocFormTaskService.generateDocument(taskId)` | 提交确认后的字段，生成填写后的文档 |
| 3 | 获取参考文档 | `ILingdocFormTaskService.findReferences(taskId)` | 从 `lingdoc_file_index` 中检索与任务相关的参考文档 |

---

## 3. 接口详情

### 3.1 识别表格字段

**用途**：将用户上传的空白表格发送给 AI，AI 解析表格结构，识别所有可填写字段，并尝试从知识库文档中提取建议值。

**调用时机**：`LingdocFormController.upload()` 保存文件并创建任务后，异步调用此方法。

**输入参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `taskId` | String | 是 | 任务ID，用于关联和日志追踪 |
| `filePath` | String | 是 | 上传文件的物理存储路径 |
| `fileType` | String | 是 | 文件类型（pdf/doc/docx/xls/xlsx） |
| `referenceDocIds` | List<String> | 否 | 参考文档ID列表（来自 `lingdoc_file_index.file_id`），AI 会优先从这些文档中提取信息 |

#### 字段类型枚举

| fieldType | 说明 | 示例 |
|-----------|------|------|
| `text` | 纯文本 | 姓名、地址 |

| code | 说明 | 处理建议 |

| 5004 | 参考文档读取失败 | 跳过该文档，继续识别 |

### 3.2 生成填写后文档

**用途**：用户确认字段值后，调用此接口生成填写完成的文档。

**调用时机**：用户点击"AI 填写"按钮，`LingdocFormController.generate()` 中调用。

**输入参数**：

```json
{
  "taskId": "task_abc123",
  "originalFileUrl": "/upload/lingdoc/form/2026/04/19/file_original_001_奖学金申请表.pdf",
  "fields": [
    {
      "fieldName": "姓名",
      "fieldValue": "张三",
      "fieldType": "text"
    },
    {
      "fieldName": "学号",
      "fieldValue": "2023001001",
      "fieldType": "text"
    },
    {
      "fieldName": "申请日期",
      "fieldValue": "2026-04-19",
      "fieldType": "date"
    },
    {
      "fieldName": "申请类别",
      "fieldValue": "国家奖学金",
      "fieldType": "select"
    }
  ]
}
```

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `taskId` | String | 是 | 任务ID |
| `originalFileUrl` | String | 是 | 原始文件存储路径 |
| `fields` | List<Field> | 是 | 用户确认后的字段列表 |

#### Response

  "data": {

### 3.3 获取参考文档

**用途**：根据任务信息和关键词，从 Vault 文件索引（`lingdoc_file_index`）中检索相关参考文档，优先使用已入知识库的文件（通过 `lingdoc_file_ai_meta.kb_id IS NOT NULL` 判定）。

**调用时机**：`LingdocFormController.getInfo()` 获取任务详情时，一并返回参考文档列表。

**检索数据源**：`lingdoc_file_index` 表（`file_name`、`file_content` 字段），优先返回 `lingdoc_file_ai_meta.kb_id IS NOT NULL` 的文件。

**输入参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `taskId` | String | 是 | 任务ID |
| `keywords` | String | 否 | 关键词，逗号分隔（如：奖学金,申请,成绩） |
| `kbId` | String | 否 | 指定知识库ID检索，不指定则检索用户所有知识库 |
| `limit` | Integer | 否 | 返回数量上限，默认 10 |

#### Response

## 4. 数据模型

### 4.1 AI 字段识别结果

识别结果需转换为 `LingdocFormField` 对象列表持久化到数据库：

```java
// 对应表：lingdoc_form_field
public class LingdocFormField {
    private String fieldId;        // 字段ID（UUID）
    private String taskId;         // 所属任务ID
    private String fieldName;      // 字段名
    private String fieldType;      // 字段类型：text/date/number/select/checkbox
    private String fieldLabel;     // 原始标签
    private String aiValue;        // AI建议值
    private String userValue;      // 用户确认值
    private String isConfirmed;    // 是否已确认：0否 1是
    private BigDecimal confidence; // 置信度
    private String sourceDocId;    // 来源文件ID（lingdoc_file_index.file_id）
    private String sourceDocName;  // 来源文档名
    private Integer sortOrder;     // 排序
}
```

参考文档结果需转换为 `LingdocFormReference` 对象列表持久化：

```java
// 对应表：lingdoc_form_reference
public class LingdocFormReference {
    private String refId;        // 关联ID（UUID）
    private String taskId;       // 任务ID
    private String docId;        // 文件ID（lingdoc_file_index.file_id）
    private String docName;      // 文档名称
    private String docPath;      // 文档存储路径
    private String docType;      // 文档类型
    private BigDecimal relevance;// 相关性评分
    private String isSelected;   // 是否被选中：0否 1是
}
```

### 4.2 AI 文档生成请求

```java
public class AiGenerateRequest {
    private String taskId;
    private String originalFileUrl;    // 原始文件路径
    private List<ConfirmedField> fields;
}

public class ConfirmedField {
    private String fieldName;
    private String fieldValue;         // 用户确认后的值
    private String fieldType;
}
```

### 4.3 AI 文档生成结果

生成结果需更新到 `LingdocFormTask`：

```java
// 对应表：lingdoc_form_task
public class LingdocFormTask {
    private String taskId;
    private String filledFileId;     // 生成后文档文件ID
    private String filledFileUrl;    // 生成后文件存储路径
    private String filledFileName;   // 生成后文件名
    private String status;           // 更新为 "3"（已生成）
    private Integer fieldCount;      // 字段总数
    private Integer tokenCost;       // Token消耗总量
}
```

## 5. 协作约定

### 5.1 开发顺序（基于当前代码状态）
1. **第一步**：AI 开发者在 `ILingdocFormTaskService` 中实现 `extractFields(taskId)` 方法（返回 mock 数据）
2. **第二步**：在 `LingdocFormController.upload()` 的 TODO 处接入 extract 调用
3. **第三步**：前端页面已完整实现（`src/views/lingdoc/form/index.vue`），无需重复开发
4. **第四步**：AI 开发者实现 `generateDocument(taskId)` 方法
5. **第五步**：在 `LingdocFormController.generate()` 的 TODO 处接入 generate 调用
6. **第六步**：联调测试，AI 开发者替换 mock 为真实 AI 逻辑
