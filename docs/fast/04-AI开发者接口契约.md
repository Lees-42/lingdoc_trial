# AI 开发者接口契约 —— 表格填写助手

**文档编号**: LingDoc-FAST-004  
**版本**: v1.0  
**编制日期**: 2026-04-19  
**状态**: 草案（待另一位 AI 开发者确认）  

---

## 1. 概述

本文档定义了**表格填写助手模块**与 **AI 服务模块**之间的接口契约。两位开发者通过本文档约定的接口进行协作：
- **后端开发者（你）**：负责实现 Controller 层，调用 AI 接口，处理业务逻辑
- **AI 开发者（另一位）**：负责实现 AI 服务的内部接口，提供字段识别和文档生成能力

**通信方式**：后端服务通过 HTTP 调用 AI 服务内部接口（`@FeignClient` 或直接 `RestTemplate`）。  
**基础路径**：`http://localhost:8080/internal/ai/form`（开发环境）

---

## 2. 接口清单

| 序号 | 接口名称 | 方法 | 路径 | 说明 |
|------|----------|------|------|------|
| 1 | 识别表格字段 | POST | `/internal/ai/form/extract` | 上传空白表格，返回字段列表 |
| 2 | 生成填写后文档 | POST | `/internal/ai/form/generate` | 提交字段数据，生成填写后的文档 |
| 3 | 获取参考文档 | GET | `/internal/ai/form/references` | 根据任务获取 Vault 中的参考文档 |

---

## 3. 接口详情

### 3.1 识别表格字段

**用途**：将用户上传的空白表格发送给 AI，AI 解析表格结构，识别所有可填写字段，并尝试从参考文档中提取建议值。

```
POST /internal/ai/form/extract
Content-Type: multipart/form-data
```

#### Request

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `file` | MultipartFile | 是 | 用户上传的文本格式表格描述（TXT/MD/CSV/JSON/XML/YAML） |
| `taskId` | String | 是 | 任务ID，用于关联和日志追踪 |
| `referenceDocIds` | List<String> | 否 | 参考文档ID列表（来自 Vault），AI 会优先从这些文档中提取信息 |

#### 字段类型枚举

| fieldType | 说明 | 示例 |
|-----------|------|------|
| `text` | 纯文本 | 姓名、地址 |
| `date` | 日期 | 申请日期、出生日期 |
| `number` | 数字 | 学号、金额 |

#### 错误码

| code | 说明 | 处理建议 |

| 5004 | 参考文档读取失败 | 跳过该文档，继续识别 |

### 3.2 生成填写后文档

**用途**：用户确认字段值后，调用此接口生成填写完成的文档。

```
POST /internal/ai/form/generate
Content-Type: application/json
```

#### Request

```json
{
  "taskId": "task_abc123",
  "originalFileId": "file_original_001",
  "originalFileUrl": "/upload/lingdoc/form/2026/04/19/file_original_001_奖学金申请表.docx",
  "fields": [
    {
      "fieldName": "姓名",
      "fieldValue": "张三",
      "fieldType": "text"
    },
    {
      "fieldName": "学号",
      "fieldValue": "2023001001",
      "fieldType": "text"
    },
    {
      "fieldName": "申请日期",
      "fieldValue": "2026-04-19",
      "fieldType": "date"
    },
    {
      "fieldName": "申请类别",
      "fieldValue": "国家奖学金",
      "fieldType": "select"
    }
  ]
}
```

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `taskId` | String | 是 | 任务ID |
| `originalFileId` | String | 是 | 原始文件ID |
| `originalFileUrl` | String | 是 | 原始文件存储路径 |
| `fields` | List<Field> | 是 | 用户确认后的字段列表 |

#### Response

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "taskId": "task_abc123",
    "filledFileId": "file_filled_001",
    "filledFileUrl": "/upload/lingdoc/form/2026/04/19/filled/task_abc123_filled.docx",
    "filledFileName": "奖学金申请表_已填写_20260419.docx",
    "fieldCount": 12,
    "filledCount": 10,
    "emptyCount": 2,
    "tokenCost": 1200
  }
}
```

#### 错误码

| code | 说明 |
|------|------|
| 5101 | 原始文件不存在 |
| 5102 | 文档生成失败 |
| 5103 | 字段值格式校验失败 |

### 3.3 获取参考文档

**用途**：根据任务信息和关键词，从 Vault 中检索相关参考文档。

```
GET /internal/ai/form/references?taskId={taskId}&keywords={keywords}&limit={limit}
```

#### Request Parameters

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `taskId` | String | 是 | 任务ID |
| `keywords` | String | 否 | 关键词，逗号分隔（如：奖学金,申请,成绩） |
| `limit` | Integer | 否 | 返回数量上限，默认 10 |

#### Response

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "taskId": "task_abc123",
    "total": 5,
    "references": [
      {

      {
        "docId": "doc_def012",
```

## 4. 数据模型

### 4.1 AI 字段识别结果 (AiExtractResult)

```java
public class AiExtractResult {
    private String taskId;
    private Integer fieldCount;
    private List<AiField> fields;
    private List<AiReference> references;
    private Integer tokenCost;
}

public class AiField {
    private String fieldName;       // 字段名
    private String fieldType;       // 字段类型
    private String fieldLabel;      // 原始标签
    private String suggestedValue;  // AI建议值
    private BigDecimal confidence;  // 置信度
    private String sourceDocId;     // 来源文档ID
    private String sourceDocName;   // 来源文档名
    private Integer sortOrder;      // 排序
    private List<String> options;   // 下拉选项（仅select类型）
}

public class AiReference {
    private String docId;
    private String docName;
    private String docPath;
    private BigDecimal relevance;
    private List<String> matchedFields;
}
```

### 4.2 AI 文档生成请求 (AiGenerateRequest)

```java
public class AiGenerateRequest {
    private String taskId;
    private String originalFileId;
    private String originalFileUrl;
    private List<ConfirmedField> fields;
}

public class ConfirmedField {
    private String fieldName;
    private String fieldValue;
    private String fieldType;
}
```

### 4.3 AI 文档生成结果 (AiGenerateResult)

```java
public class AiGenerateResult {
    private String taskId;
    private String filledFileId;
    private String filledFileUrl;
    private String filledFileName;
    private Integer fieldCount;
    private Integer filledCount;
    private Integer emptyCount;
    private Integer tokenCost;
}
```

## 5. 协作约定

### 5.1 开发顺序
1. **第一步**：AI 开发者先实现 `/internal/ai/form/extract` 接口（返回 mock 数据即可）
2. **第二步**：后端开发者实现 Controller 和 Service，对接 extract 接口
3. **第三步**：前端开发者完成页面和 upload/confirm 流程
4. **第四步**：AI 开发者实现 `/internal/ai/form/generate` 接口
5. **第五步**：后端开发者对接 generate 接口，完成下载流程
6. **第六步**：联调测试，AI 开发者替换 mock 为真实 AI 逻辑

### 5.2 Mock 数据约定
在 AI 接口尚未实现前，AI 开发者提供一个 Mock Controller：
```java
@RestController
@RequestMapping("/internal/ai/form")
public class AiFormMockController {
    // 返回固定的 mock 字段数据，确保前端可正常渲染
}
```

### 5.3 沟通渠道
- 接口变更需更新本文档，并通知双方
- 每日同步接口对接进度
- 遇到阻塞问题 2 小时内响应
