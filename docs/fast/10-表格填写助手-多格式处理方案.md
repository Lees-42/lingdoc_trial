# 表格填写助手 —— 多格式文件处理方案

**文档编号**: LingDoc-FAST-010  
**版本**: v1.0  
**编制日期**: 2026-04-21  
**状态**: 定稿  

---

## 1. 问题背景

表格填写助手支持用户上传 doc、docx、pdf、xlsx、xls 格式的空白表格。但不同格式的"填写后生成"策略不同：

- **HTML**：后端可直接用 Jsoup 进行 DOM 渲染（`.blank` → `.filled`）
- **Word / PDF / Excel**：二进制格式，后端渲染需要引入 Apache POI、PDFBox 等库，复杂度高

因此需要明确：由谁来负责生成填写后的文件？

---

## 2. 方案对比

| 维度 | 方案 A：AI 直接生成 | 方案 B：后端模板渲染 | 方案 C：统一转 HTML |
|------|-------------------|---------------------|-------------------|
| **实现方** | AI 服务侧 | 后端 | 后端 + AI |
| **原理** | AI 用 python-docx/PyPDF2/openpyxl 直接修改原文件 | 后端用 POI/PDFBox 填充字段 | 先转 HTML → AI 处理 → 再转回 |
| **格式还原度** | 高 | 中（依赖原文件结构标记） | 低（转换有精度损失） |
| **后端复杂度** | 低（只保存文件） | 高（需处理多种格式库） | 中（需转换工具） |
| **AI 复杂度** | 高（需文件操作能力） | 低（只返回字段值 JSON） | 低 |

---

## 3. 决策

**采用方案 A：AI 直接生成填写后的文件**

理由：
1. 项目当前主要格式为 docx/pdf/xlsx，后端用 POI/PDFBox 精确渲染难度大（尤其是无结构标记的 PDF）
2. AI 侧通常有 Python 环境，安装 python-docx、PyPDF2、openpyxl 等库成本低
3. 后端职责最小化，只需保存 AI 生成的文件并更新任务状态
4. 保留了 HTML 的 Jsoup 渲染作为备用路径

## 4. 接口调整

### 4.1 AI 生成结果 DTO（`AiGenerateResult`）

扩展为同时支持方式 A 和方式 B：

```java
public class AiGenerateResult {
    /** 方式 A：AI 直接生成的填写后文件绝对路径 */
    private String filledFilePath;

    /** 方式 B：字段值映射表（HTML 备用渲染用） */
    private Map<String, String> filledValues;

    /** Token 消耗量 */
    private Integer tokenCost;
}
```

**使用规则**：
- 如果 `filledFilePath != null` → 后端直接使用该文件（方式 A）
- 否则 → 回退到 `filledValues` 本地渲染（方式 B，仅 HTML 支持）

### 4.2 后端生成流程（`generateDocument`）

```
1. 读取原始文件路径
2. 获取已确认字段列表
3. 调用 aiFormService.generate(...) → 返回 AiGenerateResult
4. 优先判断 result.filledFilePath：
   ├─ 不为空 → 复制 AI 生成文件到标准输出目录
   └─ 为空 → 回退本地渲染：
        ├─ HTML → Jsoup 渲染
        └─ 其他 → 复制原文件（未填写，兜底）
5. 更新任务状态为"已生成"
```

## 5. AI 开发者侧要求

### 5.1 字段识别（`extract`）

- 输入：原始文件路径 + 文件名
- 建议：将文件转为文本或图片给大模型理解表格结构
- 输出：`AiExtractResult`（字段列表 + 参考文档列表）

### 5.2 文档生成（`generate`）

- 输入：taskId + 原始文件路径 + 已确认字段列表
- 处理：
  1. 根据原始文件格式选择对应库
     - docx → `python-docx`
     - pdf → `PyPDF2` / `pypdf`（填充 AcroForm）或 `pdfplumber` + `reportlab`
     - xlsx → `openpyxl`
     - html → 直接文本替换或返回字段值映射（由后端渲染）
  2. 将 confirmedFields 中的 `userValue`（或 `aiValue`）填入对应位置
  3. 保存填写后的文件到临时路径
- 输出：`AiGenerateResult`
  - 设置 `filledFilePath` = 填写后文件的绝对路径
  - 可选同时填充 `filledValues`（供 HTML 备用）
  - 设置 `tokenCost`

### 5.3 Mock 实现参考

当前 `MockAiFormServiceImpl` 的 Mock 逻辑：
- `filledFilePath` = 原始文件路径（模拟 AI 已生成，实际未修改）
- `filledValues` = 将 confirmedFields 的值填充为映射表

真实 AI 实现替换时，只需将 `filledFilePath` 指向真实生成后的文件即可。
