# 灵档（LingDoc）快速运行教程

> 本教程面向首次运行灵档项目的开发者，提供从环境准备到前后端联调的最简路径。
> 
> **详细配置说明、多环境启动方式及完整 FAQ 请参考**：[`docs/ruoyi/05-后端迁移与启动指南.md`](../ruoyi/05-后端迁移与启动指南.md)

---

## 1. 环境检查

确保已安装以下软件：

| 软件 | 最低版本 | 说明 |
|------|---------|------|
| JDK | 17+ | 后端运行环境 |
| Maven | 3.8+ | 后端构建工具 |
| Node.js | 18+ | 前端运行环境 |
| MySQL | 8.0+ | 业务数据库 |
| Redis | 5.0+ | 缓存与会话 |

快速验证命令（Windows PowerShell）：

```powershell
java -version; mvn -v; node -v; npm -v
```

启动 Redis（项目自带 Windows 版，保持窗口运行）：

```powershell
cd redis-temp
.\redis-server.exe redis.windows.conf
```

> 确保 MySQL 服务已启动。

---

## 2. 初始化数据库

创建数据库并导入初始数据：

```sql
CREATE DATABASE `ruoyi`
  DEFAULT CHARACTER SET utf8mb4
  COLLATE utf8mb4_general_ci;
```

```powershell
cd ruoyi-server
mysql -u root -p ruoyi < sql\ry_20260321.sql
```

> 不需要定时任务时可跳过 `quartz.sql`。

---

## 3. 修改后端配置

编辑以下两个文件，替换为您的本地账号密码：

1. **`ruoyi-server/ruoyi-admin/src/main/resources/application-druid.yml`** — 修改 MySQL 连接
2. **`ruoyi-server/ruoyi-admin/src/main/resources/application.yml`** — 如有需要修改 Redis 密码

详细配置示例请参考 [`docs/ruoyi/05-后端迁移与启动指南.md`](../ruoyi/05-后端迁移与启动指南.md)。

---

## 4. 启动后端

**推荐方式**：使用 IntelliJ IDEA 打开 `ruoyi-server/`，直接运行 `RuoYiApplication.java`。

**或命令行方式**：

```powershell
cd ruoyi-server
mvn clean package -DskipTests
java -jar ruoyi-admin/target/ruoyi-admin.jar
```

验证：浏览器访问 `http://localhost:8080/captchaImage`，若能显示验证码则后端启动成功。

---

## 5. 启动前端并访问系统

在项目根目录执行：

```powershell
npm install
npm run dev
```

打开浏览器访问：`http://localhost`

默认账号：
- **账号**：`admin`
- **密码**：`admin123`

## 常见问题速查

| 问题 | 快速解决 |
|------|---------|
| 后端启动报错 "Communications link failure" | 检查 MySQL 是否启动，`application-druid.yml` 配置是否正确 |
| 后端启动报错 "Connection refused"（Redis） | 检查 Redis 是否启动，`application.yml` 密码是否正确 |
| 登录提示 "认证失败，无法访问系统资源" | 确保浏览器地址栏是 `http://localhost`（不是 `127.0.0.1`） |
| 侧边栏出现空白菜单项 | 后端 `sys_menu` 保留了前端已删除模块的菜单，可暂时忽略 |
| 8080 端口被占用 | 修改 `application.yml` 的 `server.port`，并同步修改 `vite.config.js` 的 `baseUrl` |

更多问题请参考 [`docs/ruoyi/05-后端迁移与启动指南.md`](../ruoyi/05-后端迁移与启动指南.md)。

## 下一步

系统运行成功后，您可以：
- 阅读 [`docs/spec/01-SRS-需求说明书.md`](../spec/01-SRS-需求说明书.md) 了解灵档的产品愿景与路线图
- 阅读 [`docs/ruoyi/03-新增页面与接口规范.md`](../ruoyi/03-新增页面与接口规范.md) 学习如何规范地新增功能模块
- 开始您的第一次开发联调！

**灵档四大功能简介**：
- **自动规整**：拖拽文件到上传区，系统模拟 AI 解析并生成规范命名和分类路径，确认后归档
- **灵犀问答**：在对话界面用自然语言提问，系统模拟 AI 从本地文档中检索并返回答案，支持多轮对话和历史会话
- **版本溯源**：查看已归档文件的历史版本时间线，点击任意版本可预览，支持回滚到历史版本
- **关系图谱**：以 ECharts 力导向图可视化展示本地文件与标签的关联网络，支持拖拽、缩放、搜索和设置面板调整
