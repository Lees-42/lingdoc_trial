# 当前项目结构说明

> 本文档描述的是**单体仓库（Monorepo）模式下**的完整项目骨架。当前仓库包含：精简版 RuoYi-Vue 前端 + 完整的 RuoYi-Vue 3.9.2 Spring Boot 后端。

---

## 1. 项目概述

本项目采用**前后端分离、单体仓库统一管理**的架构：
- **前端**：基于 `RuoYi-Vue3` 的精简版 UI（删除了不常用模块页面，核心基础设施完整保留）
- **后端**：标准的 `RuoYi-Vue 3.9.2` Maven 多模块项目（完整保留所有业务接口）
- **联调方式**：前端开发服务器（`localhost:80`）通过 Vite 代理将 `/dev-api` 请求转发到后端（`localhost:8080`）

### 技术栈总览

| 层级 | 技术 | 版本 | 说明 |
|------|------|------|------|
| 前端框架 | Vue | 3.5.26 | 组合式 API + `<script setup>` |

| 构建工具 | Maven | 3.8+ | 后端多模块构建 |

后端采用标准的 Maven 多模块结构，**即使前端删除了部分页面，后端的 Controller 和 Service 仍是完整的**。

**职责**：接收 HTTP 请求、参数校验、调用 Service、返回统一响应。

### 4.2 `ruoyi-system` — 系统管理业务模块

```
ruoyi-system/
└── src/main/java/com/ruoyi/system/
    ├── service/          # 业务逻辑层
    ├── mapper/           # 数据访问层（MyBatis 接口 + XML）
    ├── domain/           # 实体对象（POJO）
    └── utils/            # 模块内工具
```

**覆盖业务**：用户管理、角色管理、菜单管理、部门管理、岗位管理、字典管理、参数配置、通知公告、操作日志、登录日志。

> 虽然前端删除了角色/菜单/部门/岗位/通知公告等页面，但 `ruoyi-system` 中的完整 Service 和 Mapper 均保留，对应的 Controller 也在 `ruoyi-admin` 中正常运行。

### 4.3 `ruoyi-framework` — 框架核心支撑

**职责**：
- **安全认证**：Spring Security 配置、JWT Token 生成与解析、登录认证过滤器
- **统一响应**：全局异常处理、`AjaxResult` 统一返回格式
- **AOP 日志**：操作日志自动记录（基于注解）
- **数据源**：Druid 数据源配置、多数据源支持
- **Redis 封装**：缓存工具、分布式锁等
- **拦截器**：请求日志、重复提交拦截、XSS 过滤

### 4.4 `ruoyi-common` — 公共基础

```
ruoyi-common/
├── core/               # 核心基类、常量
├── utils/              # 通用工具类（字符串、日期、文件、反射等）

```

### 4.5 `ruoyi-generator` — 代码生成器

**状态**：前端页面 `views/tool/gen/` 已删除，但后端模块完整保留。

## 5. 前后端交互链路

以下是一条典型请求从前端到后端的完整生命周期：

```
浏览器访问 http://localhost
        ↓
前端 Vite Dev Server（端口 80）
        ↓
vite.config.js 代理 /dev-api → http://localhost:8080
        ↓
后端 ruoyi-admin（Tomcat / 端口 8080）
        ↓
Spring Security 过滤器链（JWT 校验、权限检查）
        ↓
Controller（ruoyi-admin）
        ↓
Service / Mapper（ruoyi-system 或其他模块）
        ↓
MySQL / Redis
        ↓
统一响应体（ruoyi-framework 封装）
        ↓
前端 request.js 响应拦截器处理 code / msg
        ↓
Vue 组件更新 Pinia Store 或本地状态
        ↓
页面渲染更新
```

### 代理配置说明

文件：`vite.config.js`

```javascript
const baseUrl = 'http://localhost:8080'

server: {
  proxy: {
    '/dev-api': {
      target: baseUrl,
      changeOrigin: true,
      rewrite: (p) => p.replace(/^\/dev-api/, '')
    }
  }
}
```

前端环境变量 `.env.development`：
```
VITE_APP_BASE_API = '/dev-api'
```

这意味着前端所有以 `/dev-api` 开头的 API 请求，都会被代理到后端的 `http://localhost:8080`。

## 7. 前端已删除模块的兼容说明

以下标准 RuoYi-Vue 模块的**前端页面**在当前项目中已被移除：

- **系统管理**：角色管理、菜单管理、部门管理、岗位管理、通知公告
- **监控管理**：在线用户、定时任务、服务监控、登录日志、数据监控
- **系统工具**：代码生成、系统接口（Swagger UI 页面）

### 兼容注意事项

1. **后端接口完整可用**：上述模块的后端 Controller、Service、Mapper 均完整保留在 `ruoyi-admin` 和 `ruoyi-system` 中，可通过 Swagger/Postman 直接调用。
2. **菜单数据一致性**：若后端 `sys_menu` 中仍保留已删除模块的菜单记录，这些菜单项仍会在侧边栏显示，但点击后内容区将渲染为空白页，并在浏览器控制台输出组件未找到的警告。建议同步清理后端菜单数据，或保持前后端菜单一致性。
3. **恢复模块方法**：如需恢复某个已删除的前端模块，需从官方仓库复制完整的 `views/xxx/` 页面目录和 `src/api/xxx.js` 接口文件。
