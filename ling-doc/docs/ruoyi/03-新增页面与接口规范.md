# 新增页面与接口规范

> 本文档说明如何基于当前项目的前后端对接模式，规范地新增页面、接口和状态模块。

---

## 1. 路由注册规范

### 1.1 公共路由（`constantRoutes`）
用于**不依赖权限**即可访问的页面，如登录页、注册页、错误页、首页、个人中心等。

文件：`src/router/index.js`

```javascript
export const constantRoutes = [
  {
    path: '/login',
    component: () => import('@/views/login'),
    hidden: true
  },
  {
    path: '/401',
    component: () => import('@/views/error/401'),
    hidden: true
  }
]
```

### 1.2 动态路由（`dynamicRoutes`）
用于**需要权限控制**的隐藏路由，如详情页、编辑页、字典数据页等。这些路由不在后端菜单中，因此需要在前端代码中显式注册。

```javascript
export const dynamicRoutes = [
  {
    path: '/system/dict-data',
    component: Layout,
    hidden: true,
    permissions: ['system:dict:list'],
    children: [
      {
        path: 'index/:dictId(\\d+)',
        component: () => import('@/views/system/dict/data'),
        name: 'Data',
        meta: { title: '字典数据', activeMenu: '/system/dict' }
      }
    ]
  }
]
```

> **注意区分两套路由来源**：
> - **`dynamicRoutes`**：前端 `src/router/index.js` 中硬编码的隐藏路由（用于详情页、编辑页等不在侧边栏显示的页面）
> - **后端菜单路由**：通过 `getRouters()` 接口动态获取，用于渲染侧边栏菜单树

### 1.3 路由权限字段说明

| 字段 | 用途 | 示例 |
|------|------|------|
| `roles` | 基于角色的访问控制 | `roles: ['admin']` |
| `permissions` | 基于权限标识的访问控制 | `permissions: ['system:user:list']` |
| `hidden` | 是否在侧边栏隐藏 | `hidden: true` |
| `activeMenu` | 指定高亮的侧边栏菜单 | `activeMenu: '/system/dict'` |

> 后端通过 `getRouters()` 返回的菜单数据会自动映射到 `views/` 下的 `.vue` 文件。若后端新增了菜单，只需保证 `component` 路径与 `src/views/` 下的文件路径一致即可。

## 2. API 文件创建规范

### 2.1 文件位置
- 系统管理类接口：`src/api/system/模块名.js`
- 监控管理类接口：`src/api/monitor/模块名.js`
- 灵档业务类接口：`src/api/lingdoc/模块名.js`
- 登录/菜单等通用接口：`src/api/根目录`

### 2.2 函数命名规范
- 列表查询：`listXxx(query)`
- 详情查询：`getXxx(id)`
- 新增：`addXxx(data)`
- 修改：`updateXxx(data)`
- 删除：`delXxx(id)`
- 导出/下载：`exportXxx(query)`

> 注：项目中部分遗留接口（如 `src/api/monitor/operlog.js` 中的 `list`）未完全遵循 verb-noun 规范，**新增接口须严格执行**。灵档模块当前为纯前端 mock，API 文件保留标准模板方便后续接入。

### 2.3 标准模板

```javascript
import request from '@/utils/request'

// 查询列表
export function listExample(query) {
  return request({
    url: '/system/example/list',
    method: 'get',
    params: query
  })
}

// 查询详细
export function getExample(exampleId) {
  return request({
    url: '/system/example/' + exampleId,
    method: 'get'
  })
}

// 新增
export function addExample(data) {
  return request({
    url: '/system/example',
    method: 'post',
    data: data
  })
}

// 修改
export function updateExample(data) {
  return request({
    url: '/system/example',
    method: 'put',
    data: data
  })
}

// 删除
export function delExample(exampleId) {
  return request({
    url: '/system/example/' + exampleId,
    method: 'delete'
  })
}

// 导出
export function exportExample(query) {
  return request({
    url: '/system/example/export',
    method: 'get',
    params: query,
    responseType: 'blob'
  })
}
```

**灵档模块示例**（`src/api/lingdoc/organize.js`）：

```javascript
import request from '@/utils/request'

// 查询待规整文件列表
export function listOrganize(query) {
  return request({
    url: '/lingdoc/organize/list',
    method: 'get',
    params: query
  })
}

// 确认归档
export function confirmOrganize(data) {
  return request({
    url: '/lingdoc/organize/confirm',
    method: 'post',
    data: data
  })
}
```

### 2.4 请求拦截器行为
`src/utils/request.js` 已统一处理以下逻辑：
- **Token 注入**：自动从 Cookie 读取并添加 `Authorization: Bearer xxx`
- **GET 参数序列化**：自动将 `params` 转为 URL 查询字符串
- **防重复提交**：POST/PUT 请求在 1000ms 内重复提交会被拦截
- **401 处理**：自动弹出重新登录提示框

> 若某个请求不需要 Token，可在 `headers` 中设置 `isToken: false`。

### 3.2 新增 Store 模块示例
若需新增全局状态（如系统通知未读数），参照 `dict.js` 的 Pinia Options API 模式：

```javascript
// src/store/modules/notification.js
const useNotificationStore = defineStore(
  'notification',
  {
    state: () => ({
      unreadCount: 0
    }),
    actions: {
      setUnreadCount(count) {
        this.unreadCount = count
      },
      async fetchUnreadCount() {
        const res = await getUnreadCount()
        this.unreadCount = res.data
      }
    }
  })

export default useNotificationStore
```

### 3.3 在组件中使用
```vue
<script setup>
import useNotificationStore from '@/store/modules/notification'

const notificationStore = useNotificationStore()
</script>

<template>
  <el-badge :value="notificationStore.unreadCount" />
</template>
```

## 4. 页面组件规范

### 4.1 推荐使用 `<script setup>`
```vue
<script setup>
import { ref, reactive, onMounted } from 'vue'
import { listExample, getExample } from '@/api/system/example'

const loading = ref(false)
const exampleList = ref([])
const queryParams = reactive({
  pageNum: 1,
  pageSize: 10,
  exampleName: ''
})

async function getList() {
  loading.value = true
  try {
    const res = await listExample(queryParams)
    exampleList.value = res.rows
  } catch (error) {
    console.error('获取列表失败:', error)
  } finally {
    loading.value = false
  }
}

onMounted(() => {
  getList()
})
</script>
```

### 4.2 Props 类型定义
```vue
<script setup>
const props = defineProps({
  dictType: {
    type: String,
    required: true
  },
  disabled: {
    type: Boolean,
    default: false
  }
})
</script>
```

### 4.3 条件渲染规范
优先使用 `&&` 短路与，避免多层嵌套的三元表达式。

```vue
<!-- PASS -->
<el-empty v-if="!exampleList.length && !loading" description="暂无数据" />
<el-table v-else :data="exampleList" v-loading="loading">
  ...
</el-table>

<!-- FAIL：三元嵌套地狱 -->
<!-- {loading ? <Spinner /> : error ? <Error /> : data ? <Table /> : <Empty />} -->
```

## 5. 权限控制与字典使用

### 5.1 权限控制

**权限标识前缀约定**：
- **当前 MVP 阶段**：基于 RuoYi 框架的模块继续使用 `system:`、`monitor:` 等前缀。
- **未来 LingDoc 独立业务模块**（如自动规整、自然语言检索）：使用 `lingdoc:` 前缀，如 `lingdoc:organize:list`。

当前项目支持两种权限控制方式：

#### 方式一：路由级权限（`permissions` / `roles`）
来源有两处：
1. **后端菜单路由**：后端 `getRouters()` 返回的菜单数据中自带 `perms` / `roles` 字段，用于控制侧边栏页面访问
2. **前端 `dynamicRoutes`**：在 `src/router/index.js` 中硬编码的隐藏路由权限，用于控制详情页等访问

```javascript
// 前端 dynamicRoutes 中的权限示例
permissions: ['system:example:list']
```

#### 方式二：按钮级权限（`v-hasPermi` 指令）
```vue
<el-button v-hasPermi="['system:example:add']">新增</el-button>
<el-button v-hasPermi="['system:example:edit']">修改</el-button>
<el-button v-hasPermi="['system:example:remove']">删除</el-button>
```

灵档模块权限标识示例：
```vue
<el-button v-hasPermi="['lingdoc:organize:confirm']">确认归档</el-button>
<el-button v-hasPermi="['lingdoc:search:clear']">清空历史</el-button>
<el-button v-hasPermi="['lingdoc:version:rollback']">回滚版本</el-button>
```

### 5.2 字典使用
#### 加载字典
```javascript
import { getDicts } from '@/api/system/dict/data'

const dicts = reactive({
  sys_normal_disable: []
})

async function initDict() {
  const res = await getDicts('sys_normal_disable')
  dicts.sys_normal_disable = res.data
}
```

#### 回显字典标签
```vue
<template>
  <dict-tag :options="dicts.sys_normal_disable" :value="row.status" />
</template>
```

或手动回显：
```javascript
import { selectDictLabel } from '@/utils/ruoyi'

const label = selectDictLabel(dicts.sys_normal_disable, row.status)
```

## 6. 文件上传规范

当前项目保留头像上传功能，文件上传统一使用 `request.js` 的 `FormData` 方式：

```javascript
export function uploadFile(data) {
  return request({
    url: '/common/upload',
    method: 'post',
    data: data,
    headers: {
      'Content-Type': 'multipart/form-data'
    }
  })
}
```

前端组件中：
```vue
<el-upload
  action="#"
  :http-request="customUpload"
  :before-upload="handleBeforeUpload"
>
  <el-button>上传文件</el-button>
</el-upload>
```

```javascript
async function customUpload(options) {
  const formData = new FormData()
  formData.append('file', options.file)
  try {
    const res = await uploadFile(formData)
    ElMessage.success('上传成功')
    return res
  } catch (error) {
    ElMessage.error('上传失败')
    throw error
  }
}
```
