# 自动规整 —— 端到端测试样例

> 用于手动验证前端上传 → AI 自动规整 → 查看建议结果的完整链路。

## 使用步骤

1. 启动后端（`ruoyi-server/ruoyi-admin.jar`）和前端（`npm run dev`）
2. 在前端 Inbox 页面点击"上传文件"
3. 选择本目录下的任意 `.txt` 文件
4. 点击"自动规整"按钮
5. 观察 AI 返回的分类建议、标签建议、重命名建议、摘要、关键词
6. 对照下方"预期结果参考"判断 AI 分析是否合理

### 1. 新建文本文档.txt（个人简历）

**内容特征**：
- 含"姓名、性别、年龄、电话、邮箱"等简历标准字段
- 有"教育背景、工作经历、技能、自我评价"板块

## 验证检查清单

对每个样例，检查 AI 返回结果是否满足：

- [ ] **分类路径合理**：是否与文档类型匹配（简历→个人/工作，合同→工作/法务，论文→学习/研究等）
- [ ] **标签准确**：是否提取了文档的核心主题标签
- [ ] **重命名有意义**：新文件名是否比原始文件名更易识别
- [ ] **摘要完整**：是否涵盖了文档的关键信息
- [ ] **关键词相关**：关键词是否与内容高度相关
- [ ] **置信度合理**：confidence 值是否在 0.5 以上（内容清晰的文档应高于 0.7）

## 常见问题

**Q: 为什么文件名都很随意？**
A: 模拟真实用户上传场景。用户通常不会规范命名文件，AI 需要通过内容分析而非文件名来推断文档类型。

**Q: 为什么内容中有夹杂信息（如PDF页脚、邮件签名、面试题）？**
A: 模拟用户从多处复制粘贴的场景。真实文档往往不是"干净"的，AI 需要具备从杂乱内容中提取核心信息的能力。

**Q: 预期结果只是参考吗？**
A: 是的。AI 的具体输出会因模型版本、温度参数、Prompt 调整而略有差异。只要方向正确（简历归为个人/工作类、发票归为财务类）即可认为分析成功。
